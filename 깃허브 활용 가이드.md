# 📁 LIKELION-MKTDiv / marketing-asset

> 마케팅본부의 워크플로우 자동화 코드 및 업무 자산을 통합 관리하는 저장소입니다.

---

## 이 저장소는 왜 만들었나요?

기존에는 자동화 스크립트와 코드가 **각자의 로컬 PC에 흩어져** 있었습니다.
이로 인해 발생하는 문제를 해결하기 위해 이 저장소를 운영합니다.

| 기존 문제 | GitHub 도입 후 |
|---|---|
| 담당자 퇴사 시 코드도 함께 사라짐 | 클라우드에 영구 보관, 누구나 이어서 사용 가능 |
| 누가 뭘 수정했는지 알 수 없음 | 모든 변경 이력이 자동 기록됨 |
| 최신 버전이 어디 있는지 모름 | 항상 GitHub가 최신본 (Single Source of Truth) |
| 팀원 간 코드 공유가 어려움 | 링크 하나로 열람·다운로드 가능 |

## 폴더 구조

각 담당자는 **본인 영문 이름**으로 개인 폴더를 생성하고, 그 안에서 카테고리별로 파일을 관리합니다.

```
marketing-asset/
├── hong-gildong/            # 홍길동 담당자 폴더
│   ├── python/              # Python 자동화 스크립트
│   ├── gas/                 # Google Apps Script
│   ├── sql/                 # SQL 쿼리
│   ├── web/                 # JS, HTML 등 웹 관련 코드
│   └── prompts/             # AI 프롬프트
├── kim-minjun/              # 김민준 담당자 폴더
│   ├── python/
│   ├── gas/
│   ├── sql/
│   ├── web/
│   └── prompts/
├── shared/                  # 공용 자산 (팀 공통으로 사용하는 코드·프롬프트)
│   ├── python/
│   ├── gas/
│   ├── sql/
│   ├── web/
│   └── prompts/
└── README.md                # 이 파일
```

### 폴더 생성 규칙

| 항목 | 규칙 | 예시 |
|---|---|---|
| **개인 폴더명** | 영문 이름, 소문자, 하이픈(-) 구분 | `hong-gildong` |
| **하위 카테고리** | 아래 5가지 중 필요한 것만 생성 | `python/`, `gas/`, `sql/`, `web/`, `prompts/` |
| **프로젝트 폴더** | 카테고리 안에 프로젝트명으로 생성 | `hong-gildong/python/crm-sync/` |
| **공용 자산** | 팀 전체가 쓰는 코드는 `shared/` 에 저장 | `shared/sql/monthly-report/` |

### 개인 폴더 생성 요청 방법

1. **marketing-division 슬랙 채널**에 아래 형식으로 요청
   ```
   [폴더 생성 요청] 영문이름: hong-gildong
   ```
2. 관리자가 폴더 생성 후 완료 알림
3. 이후 본인 폴더 내에서 자유롭게 파일 업로드

## 처음 세팅하기 (최초 1회)

### 1단계: GitHub 계정 만들기

1. [github.com](https://github.com) 접속 → **Sign up**
2. 이메일, 비밀번호, 사용자명 입력 후 가입 완료
3. 관리자에게 **GitHub 사용자명** 전달 → 초대 링크 수신 → 수락

### 2단계: GitHub Desktop 설치 (권장)

명령어 없이 클릭만으로 사용할 수 있는 프로그램입니다.

1. [desktop.github.com](https://desktop.github.com) 에서 다운로드 및 설치
2. GitHub 계정으로 로그인
3. **File → Clone Repository** → `LIKELION-MKTDiv/marketing-asset` 선택
4. 내 PC에 저장할 위치 지정 → **Clone** 클릭

> 이제 내 PC에 `marketing-asset` 폴더가 생겼습니다. 이 폴더 안에서 작업하면 됩니다.

## 파일 올리는 방법

### 방법 A: GitHub Desktop 사용 (권장)

```
① 내 PC의 marketing-asset 폴더에 파일 추가/수정
② GitHub Desktop 열기 → 변경된 파일 목록 자동 표시
③ 왼쪽 하단 Summary에 변경 내용 한줄 요약 입력
   예) "CRM 동기화 스크립트 v2 업로드"
④ [Commit to main] 버튼 클릭
⑤ 상단 [Push origin] 버튼 클릭 → 완료!
```

### 방법 B: GitHub 웹에서 직접 업로드

1. [이 저장소](https://github.com/LIKELION-MKTDiv/marketing-asset) 접속
2. 업로드할 폴더로 이동
3. **Add file → Upload files** 클릭
4. 파일 드래그 앤 드롭
5. 하단 "Commit changes"에 변경 내용 한줄 요약 입력
6. **Commit changes** 버튼 클릭 → 완료!

> **어떤 방법을 쓰든 결과는 동일합니다.** 편한 방법을 선택하세요.

## 커밋 메시지 작성 규칙

파일을 올릴 때 작성하는 한줄 요약(커밋 메시지)은 아래 형식을 따라 주세요.

```
[카테고리] 작업내용 요약
```

**예시:**

| 커밋 메시지 | 설명 |
|---|---|
| `[python] CRM 동기화 스크립트 최초 업로드` | 새 파일 추가 |
| `[sql] 월간 리포트 쿼리 날짜 조건 수정` | 기존 파일 수정 |
| `[prompts] 블로그 글 생성 프롬프트 v2 업데이트` | 버전 업데이트 |
| `[gas] 더 이상 사용하지 않는 폼 스크립트 삭제` | 파일 삭제 |

## 주의사항

### ⛔ 절대 올리면 안 되는 것

- **API 키, 비밀번호, 토큰** 등 인증 정보

## 자주 묻는 질문

**Q. 같은 파일을 두 사람이 동시에 수정하면 어떻게 되나요?**
A. GitHub가 "충돌(Conflict)"이 발생했다고 알려줍니다. 당황하지 마시고 관리자에게 알려주세요. 함께 해결할 수 있습니다.

**Q. 실수로 파일을 삭제하거나 잘못 수정했어요.**
A. GitHub에는 모든 변경 이력이 남아 있으므로 이전 버전으로 되돌릴 수 있습니다. 관리자에게 요청해 주세요.

**Q. 프롬프트 같은 텍스트 파일은 어떤 형식으로 올리나요?**
A. `.md`(마크다운) 또는 `.txt` 파일로 저장해서 올려주세요. GitHub에서 바로 미리보기가 가능합니다.

**Q. 용량이 큰 파일(영상, 대용량 데이터)도 올릴 수 있나요?**
A. GitHub는 파일당 100MB 제한이 있습니다. 대용량 파일은 Google Drive 링크를 README나 `.md` 파일에 기록하는 방식을 권장합니다.
