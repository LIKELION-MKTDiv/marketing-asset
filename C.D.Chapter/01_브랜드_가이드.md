## 보이스 & 톤

### 우리의 목소리

| 우리는 | 우리는 아니다 |
|---|---|
| 명확하고 솔직하다 | 모호하거나 에둘러 말하지 않는다 |
| 고객 중심이다 | 자사 중심으로 말하지 않는다 |
| 전문적이지만 친근하다 | 딱딱하거나 거리감 있게 말하지 않는다 |
| 근거 기반이다 | 감정적이거나 과장하지 않는다 |

### 상황별 톤

**고객 커뮤니케이션**
- 공손하되 너무 형식적이지 않게
- 행동을 유도하는 명확한 문장
- 전문 용어 최소화, 쉬운 언어 우선

**내부 문서**
- 간결하고 실용적으로
- 배경 설명보다 핵심 먼저
- 불필요한 경어 생략 가능

**보고서 / 제안서**
- 데이터와 근거 명시
- 결론을 먼저, 상세 내용은 뒤에

### 자주 쓰는 표현 vs. 피할 표현

| 피할 표현 | 대신 쓸 표현 |
|---|---|
| ~인 것 같습니다 | ~입니다 |
| ~을 하도록 하겠습니다 | ~하겠습니다 |
| 많은 도움이 되셨으면 합니다 | (삭제) |
| 검토 부탁드립니다 | 확인해주세요 |

## 용어사전

Claude 및 콘텐츠 작업 시 이 문서 기준으로 용어를 통일하세요.

### AI / 기술 용어

| 한국어 | 영어 | 정의 |
|---|---|---|
| AI 전환 | AX (AI Transformation) | AI를 조직 문화와 업무에 내재화하는 변화 |
| 프롬프트 | Prompt | AI에게 주는 지시문 또는 질문 |
| 컨텍스트 | Context | AI가 참고하는 배경 정보 |
| 브랜치 | Branch | Git에서 독립적인 작업 공간 |
| PR | Pull Request | 변경사항을 팀에 공유하고 검토받는 과정 |
| 커밋 | Commit | 변경사항을 Git에 저장하는 행위 |
| 플레이북 | Playbook | 특정 업무를 AI와 함께 수행하는 방법 가이드 |

### 비즈니스 용어

| 용어 | 정의 |
|---|---|
| C.D.Chapter | Customer Decision Chapter — 이 조직의 공식 명칭 |
| CDE | Customer Decision Execution 팀 약자 |
| AX | AI Experience / AI Transformation — 우리 조직의 AI 문화 지향점 |
