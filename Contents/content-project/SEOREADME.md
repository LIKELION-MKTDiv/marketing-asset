# SEOREADME.md

> 블로그 콘텐츠 작성 시 프롬프트에 함께 적용하는 콘텐츠 SEO 구조 규약
> 
> 대상 플랫폼: **네이버 블로그** | **브런치** | **인블로그(Inblog)**
> 
> 최종 수정: 2026-03-31

---

## 1. 이 문서의 목적

블로그 콘텐츠를 작성할 때, 플랫폼에 관계없이 **검색 노출에 유리한 구조**를 일관되게 유지하기 위한 내부 규약이다. AI 프롬프트에 이 문서를 함께 첨부하면 콘텐츠 SEO 품질의 편차를 줄일 수 있다.

---

## 2. 공통 콘텐츠 SEO 구조 규칙 (플랫폼 무관)

### 2-1. 헤딩 태그 계층 구조

| 태그 | 역할 | 작성 원칙 |
|------|------|----------|
| H1 (제목) | 페이지당 1개. 검색 결과에서 클릭을 유도하는 최상위 제목 | 핵심 키워드 1개 필수 포함. 질문형 또는 숫자형이 CTR에 유리. 40~60자 권장 |
| H2 (부제목) | 콘텐츠의 대주제를 구분하는 섹션 헤더 | 키워드 변형 또는 연관 키워드 포함. 1개 콘텐츠에 2~4개 |
| H3 (소제목) | H2 하위의 세부 논점 | 독자의 실제 질문/고민을 소제목으로 활용하면 구글 '관련 질문' 스니펫 노출에 유리 |
| 본문 텍스트 | 각 H3 아래 실제 내용 | 1개 단락 3~5문장. 첫 문장에 해당 섹션의 핵심 메시지 배치 |

**원칙**: H1 → H2 → H3 순서를 건너뛰지 않는다 (H1 다음에 바로 H3 사용 금지).

### 2-2. 키워드 배치 규칙

| 위치 | 배치 방법 |
|------|----------|
| H1 (제목) | 메인 키워드 1회 자연 삽입 |
| H2/H3 | 연관 키워드 또는 롱테일 키워드를 소제목에 포함 |
| 리드문 (첫 150자) | 메인 키워드 1회 + 독자 페인포인트 언급 |
| 본문 전체 | 메인 키워드 밀도 2~3% 유지. 동의어/유사어로 자연스럽게 분산 |
| 이미지 ALT 텍스트 | 키워드 포함한 이미지 설명 (네이버·인블로그에서 특히 중요) |

**금지사항**: 키워드 스터핑 (같은 키워드를 부자연스럽게 반복), 숨김 텍스트, 무관한 인기 키워드 삽입

### 2-3. 콘텐츠 구조 템플릿

```
[H1] 제목 — 핵심 키워드 + 클릭 유도 (질문형/숫자형)
  [리드문] 150자 내외. 독자 고민 공감 + 이 글에서 얻을 수 있는 것 요약

  [H2] 대주제 1
    [H3] 세부 논점 A — 독자 질문 형태 권장
      본문 텍스트 (3~5문장 단락 × 2~3개)
      (이미지 제안: 맥락에 맞는 시각 자료)
    [H3] 세부 논점 B
      본문 텍스트
  
  [H2] 대주제 2
    [H3] 세부 논점 C
      본문 텍스트

  [마무리 섹션]
    핵심 메시지 요약 + CTA (전환 유도 링크)
```

### 2-4. 본문 작성 원칙

| 항목 | 규칙 |
|------|------|
| 분량 | 1,500~3,000자 (네이버 VIEW 탭 기준 최소 1,500자 이상 권장) |
| 단락 길이 | 3~5문장. 모바일 가독성을 위해 짧게 끊기 |
| 문장 길이 | 1문장 40~60자 이내. 접속사로 연결된 장문 지양 |
| 톤 | 해요체(기본). 플랫폼별 톤앤매너는 섹션 3 참조 |
| 강조 | 섹션당 볼드(**) 1~2회만 사용. 남용 시 가독성 저하 |
| 리스트 | 3개 이상 나열 시 불릿 리스트 활용. 5개 이하로 제한 |
| 인용/콜아웃 | 핵심 요약이나 데이터 포인트에 인용 블록(>) 사용 |

### 2-5. 이미지 규칙

| 항목 | 규칙 |
|------|------|
| 배치 빈도 | H3 섹션마다 최소 1장. 스크롤 리듬 형성 |
| ALT 텍스트 | 키워드 포함 + 이미지 내용 설명 (20~50자) |
| 파일명 | 한글 또는 영문 키워드 포함 (예: 그로스마케팅-프로젝트-발표.jpg) |
| 포맷 | WebP 우선, JPG/PNG 가능. 용량 500KB 이하 권장 |
| 역할 유형 | 공감 사진, 데이터 인포그래픽, 프로세스 다이어그램, 실제 작업 화면 중 택 |

### 2-6. 내부/외부 링크

| 항목 | 규칙 |
|------|------|
| 내부 링크 | 본문 내 자사 관련 콘텐츠 1~3개 자연 삽입 |
| 외부 링크 | 신뢰도 높은 출처(공식 사이트, 보도자료) 1~2개 |
| CTA 링크 | 마무리 섹션에 1개. 볼드 + 이모지로 시각적 강조 |
| 네이버 특이사항 | 네이버 생태계 내부 링크(블로그, 포스트, 지식iN) 우선 |

## 3. 플랫폼별 SEO 특성 및 적용 규칙

### 3-1. 네이버 블로그

| 항목 | 특성 및 규칙 |
|------|------------|
| 주요 검색엔진 | 네이버 (VIEW 탭 노출이 핵심) |
| 핵심 알고리즘 | C-Rank (주제 전문성·일관성) + D.I.A (사용자 반응 기반 정보 품질) |
| 제목 | 핵심 키워드 앞배치. 80~100자 이내 (네이버 에디터 기준 최대 4,000자이나 간결 권장) |
| 본문 분량 | 최소 1,500자, 권장 2,000~2,500자 |
| 키워드 밀도 | 메인 키워드 본문 내 3~5회 자연 삽입. 제목에 1회 |
| 소제목 활용 | 네이버 에디터의 '소제목' 서식 사용 → H2/H3으로 파싱됨 |
| 이미지 | 텍스트 > 이미지. 정보를 이미지로만 담지 말 것 (검색봇이 텍스트 우선 파싱) |
| 태그 | 게시 시 관련 태그 5~10개 설정. 메인/연관 키워드 포함 |
| 채널 최적화 | 1주제 집중 블로그 운영 (C-Rank 주제 일관성). 정기 발행 (활성화 채널 우대) |
| 상호작용 | 댓글 관리, 공감/이웃 활동이 VIEW 노출에 영향 |
| 주의사항 | 인기 키워드 억지 삽입 → 스팸 분류 위험. 대가성 포스트 미표기 → 패널티 |

**네이버용 콘텐츠 작성 시 추가 체크리스트**:
- [ ] 네이버 키워드 도구에서 검색량/경쟁도 확인했는가
- [ ] 검색 결과 상위 5개 콘텐츠의 구조/분량/키워드를 벤치마킹했는가
- [ ] 네이버 서치어드바이저에 블로그 등록 + 사이트맵/RSS 제출했는가
- [ ] 이미지에 ALT 텍스트를 설정했는가
- [ ] 중복 콘텐츠(다른 플랫폼과 동일 본문)를 피했는가

### 3-2. 브런치 (Brunch)

| 항목 | 특성 및 규칙 |
|------|------------|
| 주요 검색엔진 | 구글 (브런치 도메인 자체의 DA가 높아 구글 노출에 유리) + 카카오 |
| SEO 특성 | 브런치 자체가 HTTPS, 모바일 반응형, 빠른 로딩을 기본 지원 → 테크니컬 SEO 자동 처리 |
| 제목 | 구글 타이틀 태그로 반영됨. 60자 이내 권장. 핵심 키워드 앞배치 |
| 부제목 | 메타 디스크립션 역할. 120자 이내. 키워드 + 클릭 유도 문구 |
| 본문 분량 | 2,000~3,500자 권장. 브런치 독자는 긴 글에 익숙 |
| 소제목 | 브런치 에디터의 '소제목' 서식 = H2. '중간 제목' 서식 활용 |
| 이미지 | 글 상단 대표 이미지 필수 (SNS 공유 시 썸네일). 섹션 사이 이미지로 스크롤 리듬 형성 |
| 키워드 태그 | 게시 시 '키워드' 3~5개 설정. 브런치 내부 검색 + 구글 태그 반영 |
| 매거진 기능 | 같은 주제의 글을 매거진으로 묶으면 주제 관련성 강화 (구글 토픽 클러스터와 유사 효과) |
| 백링크 활용 | 브런치 글 URL을 타 채널(링크드인, 슬랙, 뉴스레터)에 공유 → 백링크 확보 → 구글 DA 상승 |
| 주의사항 | 브런치 → 네이버 블로그 동일 본문 복사 게시 시 중복 콘텐츠 페널티. 핵심만 추려 50% 미만 유사도로 변형 필요 |

**브런치용 콘텐츠 작성 시 추가 체크리스트**:
- [ ] 제목 60자 이내 + 부제목 120자 이내를 지켰는가
- [ ] 대표 이미지를 설정했는가 (SNS 공유 썸네일)
- [ ] 키워드 태그 3~5개를 설정했는가
- [ ] 관련 매거진에 글을 포함시켰는가
- [ ] 발행 후 링크드인/슬랙 등 외부 채널에 공유했는가

### 3-3. 인블로그 (Inblog)

| 항목 | 특성 및 규칙 |
|------|------------|
| 주요 검색엔진 | 구글 (구글 Lighthouse SEO 만점 기록 플랫폼) + AI 검색엔진 (AEO/GEO 대응) |
| SEO 특성 | 테크니컬 SEO 자동 처리 (사이트맵 자동 생성, 캐노니컬 URL, 301 리디렉트, SSL, JSON-LD 스키마, robots.txt, H1~H6 자동 구조화) |
| 도메인 | 커스텀 도메인 연결 필수 (서브디렉토리 방식 권장: example.com/blog). 자사 도메인의 DA 축적에 유리 |
| 제목 | 타이틀 태그로 직접 반영됨. 60자 이내. E-E-A-T 기준 충족하는 전문성 있는 제목 |
| 메타 디스크립션 | 인블로그 에디터에서 직접 입력 가능. 120~150자. 키워드 + 가치 제안 |
| 본문 분량 | 2,000~4,000자. 구글 E-E-A-T 기준상 깊이 있는 콘텐츠 우대 |
| 헤딩 구조 | H1~H6 자동 태그 적용 → 마크다운 헤딩(#, ##, ###)을 정확히 사용해야 함 |
| 내부 링크 | 인블로그 에디터에서 블로그 내 관련 글 연결 기능 지원 → 적극 활용 |
| CTA/리드폼 | CTA 버튼 삽입 기능 제공. 콘텐츠 하단에 리드폼 배치 가능 → 전환 추적에 유리 |
| 분석 | 구글 서치콘솔 + GA 연동. 키워드별 유입/노출/게재순위 실시간 확인 |
| AEO 대응 | AI 검색엔진(ChatGPT, Perplexity 등)에 답변 소스로 선택되려면: FAQ 구조, 명확한 정의문, 구조화된 데이터(JSON-LD) 활용 |
| 주의사항 | 노션 임베딩 시 원본 노션 페이지가 검색에 중복 노출되지 않도록 노션 측 SEO 설정 확인 |

**인블로그용 콘텐츠 작성 시 추가 체크리스트**:
- [ ] 커스텀 도메인이 연결되어 있는가
- [ ] 구글 서치콘솔에 사이트맵이 제출되어 있는가
- [ ] 메타 디스크립션을 직접 입력했는가 (자동 생성에 의존하지 않기)
- [ ] H1 → H2 → H3 계층 구조를 정확히 지켰는가
- [ ] 내부 링크를 1~3개 삽입했는가
- [ ] CTA 버튼 또는 리드폼을 배치했는가
- [ ] JSON-LD 스키마 마크업을 설정했는가 (해당 시)

## 4. 플랫폼 간 교차 발행 규칙

같은 콘텐츠를 여러 플랫폼에 게시할 때 중복 콘텐츠 페널티를 피하기 위한 규칙.

| 규칙 | 설명 |
|------|------|
| 원본 지정 | 1개 플랫폼을 정본(canonical)으로 지정. 인블로그(자사 도메인) > 브런치 > 네이버 순 권장 |
| 변형 발행 | 타 플랫폼에는 핵심만 추려 50% 미만 유사도로 재작성. "전문은 [원본 링크]에서 확인하세요" 유도 |
| 발행 시차 | 원본 먼저 발행 → 구글 색인 확인(1~3일) → 변형본 발행 |
| 백링크 설계 | 변형본 본문에 원본 URL 삽입 → 원본 도메인의 DA 강화 효과 |

## 5. 콘텐츠 유형별 구조 가이드

### 5-1. 탐색 유저 대상 (고민 해소/전환 유도형)

```
[H1] 타겟의 핵심 질문 (질문형 제목)
  [리드문] 고민 공감 + 이 글에서 답을 드립니다

  [H3] 페인포인트 1순위 — "질문 형태 소제목" + 해결 근거 제시
  [H3] 페인포인트 2순위 — 데이터/사례로 불안 해소
  [H3] 페인포인트 3순위 — 운영 방식/지원 체계로 신뢰 형성

  [마무리] 핵심 요약 + CTA
```

### 5-2. 정보 탐색형 (How-to / 가이드)

```
[H1] "~하는 방법" 또는 "~가이드" (숫자형 제목)
  [리드문] 이 글에서 다루는 범위 요약

  [H2] 대주제 1 — 기초/배경
    [H3] 세부 단계 A
    [H3] 세부 단계 B
  [H2] 대주제 2 — 실행/응용
    [H3] 세부 단계 C

  [마무리] 핵심 체크리스트 요약 + 관련 콘텐츠 링크
```

### 5-3. 후기/사례형 (수강 후기, 사용 사례)

```
[H1] 구체적 결과 + 과정명/서비스명
  [리드문] 핵심 성과 1줄 요약

  [H3] Before — 시작 전 상황/고민
  [H3] During — 과정 중 경험 (구체적 에피소드)
  [H3] After — 완료 후 변화/성과 (정량 데이터 포함)

  [마무리] 추천 대상 명시 + CTA
```

## 6. 프롬프트 적용 방법

AI에게 블로그 콘텐츠 작성을 요청할 때, 아래 형식으로 이 문서를 참조시킨다.

```
아래 규칙을 적용해서 블로그 콘텐츠를 작성해줘.

- 플랫폼: [네이버블로그 / 브런치 / 인블로그]
- 콘텐츠 유형: [탐색 유저 대상 / 정보 탐색형 / 후기 사례형]
- 메인 키워드: [______]
- 연관 키워드: [______, ______, ______]
- 타겟 독자: [______]
- 톤앤매너: [해요체 / 합니다체 / 기타]
- CTA: [수강 신청 / 상담 예약 / 없음(인지 목적)]
- 참고: SEOREADME.md 규칙 적용

[기존 원고 또는 소재]
```

## 7. 셀프 체크리스트 (발행 전 최종 점검)

| # | 체크 항목 | 확인 |
|---|----------|------|
| 1 | H1에 메인 키워드가 포함되어 있는가 | ☐ |
| 2 | H1 → H2 → H3 계층 구조가 건너뜀 없이 유지되는가 | ☐ |
| 3 | 리드문(첫 150자)에 키워드 + 독자 페인포인트가 담겨 있는가 | ☐ |
| 4 | 본문 키워드 밀도가 2~3%인가 (과다/과소 아닌지) | ☐ |
| 5 | 이미지에 ALT 텍스트를 설정했는가 | ☐ |
| 6 | 내부 링크 1~3개를 삽입했는가 | ☐ |
| 7 | CTA가 마무리 섹션에 명확히 배치되어 있는가 | ☐ |
| 8 | 모바일에서 읽었을 때 단락이 너무 길지 않은가 | ☐ |
| 9 | 타 플랫폼 교차 발행 시 유사도 50% 미만으로 변형했는가 | ☐ |
| 10 | 플랫폼별 추가 체크리스트(섹션 3)를 완료했는가 | ☐ |
