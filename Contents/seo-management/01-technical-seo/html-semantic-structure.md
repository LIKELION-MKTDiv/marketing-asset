# HTML 시맨틱 구조 원칙

> `KDT_상세페이지_생성모듈_SEO정책_v3.md`의 핵심 원칙을 채널 불문 범용 규칙으로 일반화. 순수 마크다운 블로그(네이버·브런치·인블로그)에는 헤딩 서식만 적용하면 되고, 이 문서는 **에셋/컴포넌트 기반으로 HTML을 직접 만드는 모든 채널**(홈페이지, 상세페이지, 마이크로사이트, 랜딩페이지, 인터랙티브 콘텐츠)에 적용한다.

---

## 1. 핵심 전제 — "보이는 모습"과 "코드의 의미"는 다르다

CSS(Tailwind든 무엇이든)는 어떤 태그를 쓰든 보이는 모습을 자유롭게 바꿀 수 있다. 그래서 모든 요소를 `<div>`로 만들고 스타일 클래스로만 꾸미는 실수가 쉽게 발생한다. **검색엔진은 CSS 클래스를 읽지 않는다. 태그만 읽는다.**

| 눈에 보이는 것 | 코드 | 검색엔진이 이해하는 것 |
|---|---|---|
| 큰 굵은 글씨 제목 | `<h1 class="text-4xl font-bold">` | "이 페이지 주제" ✅ |
| 똑같이 보이는 큰 굵은 글씨 | `<div class="text-4xl font-bold">` | "글자가 크네… 뭔지 모르겠다" ❌ |
| 클릭 가능한 버튼 | `<a href="/consult">` | "상담 페이지로 연결됨" ✅ |
| 클릭 가능한 이미지 배너 | `<img class="cursor-pointer">` (링크 없음) | "그림이 있네… 어디로 가는진 모른다" ❌ |

**원칙: CSS로 "모양"을 만들되, 올바른 태그로 "의미"를 부여한다.**

---

## 2. 태그 선택 기준표

| 요소 | 올바른 태그 | 절대 쓰면 안 되는 것 |
|---|---|---|
| 페이지 대제목 (페이지당 1개) | `<h1>` | `<div class="text-4xl">` |
| 섹션 제목 | `<h2>` | `<div class="text-2xl">` |
| 세부 제목 | `<h3>` | `<span class="text-xl">` |
| 본문 텍스트 | `<p>` | `<div class="text-base">` |
| 링크/CTA 버튼 | `<a href="...">` | `<div onclick="...">` |
| 콘텐츠 이미지 | `<img alt="...">` | `<div class="bg-[url(...)]">` (alt 불가) |
| 순수 장식 배경 | CSS 배경 (`bg-[url(...)]`) | — (장식은 이 방법이 맞음) |
| 섹션 구분 | `<section>` | `<div>` |
| 이미지+캡션 | `<figure>` + `<figcaption>` | `<div>` + `<div>` |
| 리스트(커리큘럼, 체크리스트 등) | `<ul>`/`<ol>` + `<li>` | `<div>` 반복 |

**블록 유형별 기본 제목 레벨 (컴포넌트 조합형 페이지):**

| 블록 유형 | 기본 제목 태그 |
|---|:---:|
| 히어로(최상단) | H1 |
| 콘텐츠 섹션(기능 소개, 후기 등) | H2 |
| 상세 항목(개별 항목, 개별 후기) | H3 |

조합 시 "H1이 2개 이상" 또는 "H1 없이 H2부터 시작"하는 경우가 없는지 반드시 확인.

## 3. 이미지 규칙 — 콘텐츠 vs 장식

| 방법 | 코드 형태 | 검색엔진이 읽는 것 | alt 가능 여부 |
|---|---|---|:---:|
| `<img>` 태그 | `<img src="..." alt="설명">` | 이미지 + 설명 | ✅ |
| CSS 배경 | `<div class="bg-[url(...)]">` | 아무것도 안 보임 | ❌ |

**구분 기준**:
- **콘텐츠를 담은 이미지** (제품/서비스 화면, 후기 카드, 인물 사진, 도표, 커리큘럼) → `<img>` + alt 필수
- **순수 장식** (배경 패턴, 그라데이션, 구분선) → CSS 배경 가능

alt 텍스트: 20~50자, 키워드 포함 + 실제 이미지 내용 설명. 빈 alt 방치 금지.

## 4. 모바일 반응형과 SEO

| 클래스 패턴(Tailwind 예시) | 의미 | SEO 영향 |
|---|---|---|
| `hidden md:block` (장식/보조 요소) | 모바일에서 숨기고 PC에서만 표시 | 장식이면 문제없음 |
| `hidden md:block` (제목/본문/CTA) | 핵심 텍스트를 모바일에서 숨김 | ⚠️ 금지 — 숨겨진 텍스트는 검색엔진이 가치를 낮게 평가 |
| `text-sm md:text-lg` | 크기만 조절 | ✅ 문제없음 — 콘텐츠는 항상 존재 |

**원칙: 대제목·핵심 텍스트·CTA는 어떤 화면 크기에서도 코드에 존재해야 한다.** 크기·레이아웃 조절은 자유, "존재 자체를 지우는 것"만 금지.

## 5. 검증 예시 — 좋은 구조 vs 나쁜 구조

**검색엔진이 잘 읽는 구조:**
```html
<section>
  <img src="에셋.webp" alt="설명글">
  <h2>핵심 기능 소개</h2>
  <p>구체적인 설명 텍스트...</p>
  <a href="/consult">무료 상담 신청</a>
</section>
```

**검색엔진이 못 읽는 구조 (화면에는 똑같이 보임):**
```html
<div>
  <div style="background: url(에셋.webp)">
    <div class="text-2xl">핵심 기능 소개</div>
    <div>구체적인 설명 텍스트...</div>
    <div onclick="...">무료 상담 신청</div>
  </div>
</div>
```

두 코드는 화면에는 동일하게 보이지만 검색엔진이 읽는 정보량은 완전히 다르다.

## 6. 인터랙티브 콘텐츠(진단기, 시뮬레이터 등)에 적용 시

`interactive-builder` 에이전트가 만드는 단일 HTML 파일(유전질환 검색기 등)도 이 원칙을 그대로 따른다:
- 결과 화면의 제목/설명도 `<h1~h3>`/`<p>` 사용 (JS로 동적 렌더링해도 태그는 시맨틱하게)
- 진단 결과 카드가 이미지 기반이면 `<img alt>` 또는 SVG+`<title>`
- 공유 가능한 페이지라면 head에 og:title/description 포함
