# SEO 전체 프레임워크

> 이 문서부터 읽을 것. 테크니컬 SEO / 콘텐츠 SEO / GEO·AEO가 각각 무엇을 담당하는지, 어떤 순서로 적용하는지 정리한다.

---

## 1. 세 층위의 관계

| 층위 | 책임 | 실패하면 |
|---|---|---|
| **테크니컬 SEO** (`01-technical-seo/`) | 검색엔진이 페이지를 **크롤링·색인·이해**할 수 있게 만드는 것 | 콘텐츠가 아무리 좋아도 검색엔진이 아예 못 읽거나 못 찾음 |
| **콘텐츠 SEO** (`02-content-seo/`) | 색인된 페이지가 **검색 의도에 맞고 상위 노출될 만큼 좋은 콘텐츠**인지 | 검색엔진은 읽지만 순위가 낮음 → 트래픽 없음 |
| **GEO/AEO** (`03-geo-aeo/`) | 챗GPT·퍼플렉시티·구글 AI 개요 같은 **생성형 답변 엔진이 우리 콘텐츠를 답변 소스로 인용**하게 만드는 것 | 전통 검색에는 뜨지만 AI 답변에는 인용되지 않음 |

세 층위는 순서대로 쌓인다: 테크니컬 SEO가 안 되면 콘텐츠 SEO도 무의미하고, 콘텐츠 SEO 기본기가 없으면 GEO/AEO도 작동하지 않는다(AI 답변 엔진도 결국 잘 구조화된 콘텐츠를 우선 인용한다).

---

## 2. 판단 순서 — 새 채널/상황을 만났을 때

```
1. 이 채널이 검색엔진에 색인되는가?
   ├─ Yes (블로그, 홈페이지, 상세페이지, 마이크로사이트) → 01+02+03 모두 적용
   └─ No, 플랫폼 자체 알고리즘만 있음 (인스타그램, 유튜브, 틱톡)
        → 01(테크니컬)은 "플랫폼 자체 SEO 요소"로 축소 적용
        → 02(콘텐츠)는 "플랫폼 검색·추천 알고리즘 최적화"로 치환
        → 03(GEO/AEO)은 유튜브에 한해 부분 적용(구글 검색에 영상이 노출되므로)

2. 이 채널의 콘텐츠가 텍스트를 HTML로 직접 출력하는가, 이미지/에셋 위에 텍스트를 얹는 구조인가?
   ├─ 순수 HTML/마크다운 (블로그 플랫폼 대부분) → 플랫폼 에디터의 헤딩 서식만 정확히 쓰면 됨
   └─ 이미지 에셋 기반 모듈 (상세페이지, 마이크로사이트, SNS 카드뉴스)
        → `01-technical-seo/html-semantic-structure.md`의 "에셋+텍스트 레이어" 원칙 필수 적용
        → 콘텐츠 이미지는 <img>+alt, 장식 이미지만 CSS 배경 허용

3. 이 채널이 정본(canonical)인가, 파생 채널인가?
   → `02-content-seo/cross-publishing.md`의 정본 지정·중복 콘텐츠 회피 규칙 적용

4. 발행 전에 `05-audit/audit-checklist.md`로 최종 점검.
```

## 4. 핵심 원칙 요약 (각 문서의 1줄 결론)

- **테크니컬 SEO**: 눈에 보이는 모양(CSS)과 검색엔진이 읽는 의미(HTML 태그)는 다르다. `<div>`가 아니라 `<h1>`, `<img alt>`, `<a href>`, `<section>`으로 의미를 부여해야 한다.
- **콘텐츠 SEO**: 검색 의도에 맞는 구조(H1→H2→H3 계층, 키워드 자연 배치, 스캔 가능한 본문)와 신뢰 신호(E-E-A-T, 출처)가 순위를 결정한다.
- **GEO/AEO**: AI 답변 엔진은 "질문에 바로 답하는 명확한 문단 + 구조화된 데이터(FAQ, 정의문, 표)"를 인용 소스로 선호한다. 전통 SEO 위에 답변 가능성을 얹는 개념.
