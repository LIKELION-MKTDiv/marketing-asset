# 콘텐츠 SEO 공통 작성 규칙 (채널 불문)

> `content-project/SEOREADME.md`의 공통 규칙을 원본으로 하되, 특정 블로그 플랫폼(네이버·브런치·인블로그) 종속 내용을 제거하고 **모든 텍스트 콘텐츠 채널**(블로그, 홈페이지, 상세페이지, SNS 캡션까지)에 적용 가능한 원칙으로 일반화했다. 플랫폼별 세부 수치·기능은 `../04-channel-playbooks/`에서 각각 다룬다.

---

## 1. 헤딩 계층 구조

| 태그 | 역할 | 작성 원칙 |
|---|---|---|
| H1(제목) | 페이지/글당 1개. 검색 결과 클릭을 유도하는 최상위 제목 | 핵심 키워드 1개 필수 포함. 질문형·숫자형이 CTR에 유리. 40~60자 권장 |
| H2(부제목) | 대주제 구분 섹션 헤더 | 키워드 변형/연관 키워드 포함. 콘텐츠당 2~4개 |
| H3(소제목) | H2 하위 세부 논점 | 독자의 실제 질문/고민을 소제목으로 쓰면 AI 답변·스니펫 노출에 유리 |
| 본문 텍스트 | 각 H3 아래 실제 내용 | 1단락 3~5문장. 첫 문장에 핵심 메시지 배치 |

**원칙: H1 → H2 → H3 순서를 건너뛰지 않는다.**

---

## 2. 키워드 배치 규칙

| 위치 | 배치 방법 |
|---|---|
| H1(제목) | 메인 키워드 1회 자연 삽입 |
| H2/H3 | 연관 키워드 또는 롱테일 키워드 포함 |
| 리드문(첫 150자) | 메인 키워드 1회 + 독자 페인포인트 언급 |
| 본문 전체 | 메인 키워드 밀도 2~3% 유지, 동의어/유사어로 자연 분산 |
| 이미지 ALT 텍스트 | 키워드 포함한 이미지 설명 |

**금지사항**: 키워드 스터핑(부자연스러운 반복), 숨김 텍스트, 무관한 인기 키워드 삽입.

---

## 3. 콘텐츠 구조 템플릿 (범용)

```
[H1] 제목 — 핵심 키워드 + 클릭 유도(질문형/숫자형)
  [리드문] 150자 내외. 독자 고민 공감 + 이 글에서 얻을 수 있는 것 요약

  [H2] 대주제 1
    [H3] 세부 논점 A — 독자 질문 형태 권장
      본문 텍스트(3~5문장 단락 × 2~3개)
      (이미지 제안: 맥락에 맞는 시각 자료)
    [H3] 세부 논점 B

  [H2] 대주제 2
    [H3] 세부 논점 C

  [마무리 섹션]
    핵심 메시지 요약 + CTA
```

---

## 4. 본문 작성 원칙

| 항목 | 규칙 |
|---|---|
| 분량 | 채널별 상이(플레이북 참조). 정보형 콘텐츠 일반 기준 1,500자 이상 |
| 단락 길이 | 3~5문장. 모바일 가독성을 위해 짧게 끊기 |
| 문장 길이 | 1문장 40~60자 이내. 접속사로 이어지는 장문 지양 |
| 강조 | 섹션당 볼드 1~2회만. 남용 시 가독성 저하 |
| 리스트 | 3개 이상 나열 시 불릿 리스트, 5개 이하로 제한 |
| 인용/콜아웃 | 핵심 요약·데이터 포인트는 인용 블록(`>`) 사용 |

---

## 5. 이미지 규칙

| 항목 | 규칙 |
|---|---|
| 배치 빈도 | H3 섹션마다 최소 1장 — 스크롤 리듬 형성 |
| ALT 텍스트 | 키워드 포함 + 이미지 내용 설명(20~50자) |
| 파일명 | 한글 또는 영문 키워드 포함 |
| 포맷 | WebP 우선, 용량 500KB 이하 권장 |
| 역할 유형 | 공감 사진 / 데이터 인포그래픽 / 프로세스 다이어그램 / 실제 화면 중 택 |

---

## 6. 내부/외부 링크

| 항목 | 규칙 |
|---|---|
| 내부 링크 | 관련 자사 콘텐츠 1~3개 자연 삽입 |
| 외부 링크 | 신뢰도 높은 출처(공식 사이트, 보도자료) 1~2개 |
| CTA 링크 | 마무리 섹션에 1개. 시각적으로 강조 |

## 7. 콘텐츠 유형별 구조 가이드

### 7-1. 탐색 유저 대상 (고민 해소/전환 유도형)
```
[H1] 타겟의 핵심 질문(질문형 제목)
  [리드문] 고민 공감 + 이 글에서 답을 드립니다

  [H3] 페인포인트 1순위 — "질문 형태 소제목" + 해결 근거 제시
  [H3] 페인포인트 2순위 — 데이터/사례로 불안 해소
  [H3] 페인포인트 3순위 — 운영 방식/지원 체계로 신뢰 형성

  [마무리] 핵심 요약 + CTA
```

### 7-2. 정보 탐색형 (How-to / 가이드)
```
[H1] "~하는 방법" 또는 "~가이드"(숫자형 제목)
  [리드문] 이 글에서 다루는 범위 요약

  [H2] 대주제 1 — 기초/배경
  [H2] 대주제 2 — 실행/응용

  [마무리] 핵심 체크리스트 요약 + 관련 콘텐츠 링크
```

### 7-3. 후기/사례형
```
[H1] 구체적 결과 + 과정명/서비스명
  [리드문] 핵심 성과 1줄 요약

  [H3] Before — 시작 전 상황/고민
  [H3] During — 과정 중 경험(구체적 에피소드)
  [H3] After — 완료 후 변화/성과(정량 데이터 포함)

  [마무리] 추천 대상 명시 + CTA
```

---

## 8. E-E-A-T (경험·전문성·권위·신뢰) 신호

Google이 "사람을 위한, 도움이 되는 콘텐츠"를 명시적으로 우대(`Creating Helpful, Reliable, People-First Content`)하므로 모든 콘텐츠에 다음을 반영:

- 실제 경험/사례 기반 서술 (일반론보다 구체적 사례)
- 수치·주장에는 출처 표기, 확인되지 않은 것은 "가설/추정"으로 명시
- 광고성 콘텐츠는 광고임을 고지 (특히 제휴/협찬)
- 작성자 또는 운영주체 정보 노출

## 9. 셀프 체크리스트 (발행 전)

| # | 항목 |
|---|---|
| 1 | H1에 메인 키워드가 포함되어 있는가 |
| 2 | H1 → H2 → H3 계층이 건너뜀 없이 유지되는가 |
| 3 | 리드문(첫 150자)에 키워드 + 페인포인트가 담겨 있는가 |
| 4 | 키워드 밀도가 2~3%인가 |
| 5 | 이미지에 ALT 텍스트를 설정했는가 |
| 6 | 내부 링크 1~3개를 삽입했는가 |
| 7 | CTA가 마무리 섹션에 명확히 배치되어 있는가 |
| 8 | 모바일에서 단락이 너무 길지 않은가 |
| 9 | 채널별 추가 규칙(`../04-channel-playbooks/`)을 확인했는가 |
| 10 | 교차 발행 시 `cross-publishing.md` 규칙을 지켰는가 |
