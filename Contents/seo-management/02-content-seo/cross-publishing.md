# 채널 간 교차 발행 규칙

> 같은 콘텐츠(또는 파생 콘텐츠)를 여러 채널에 게시할 때 중복 콘텐츠 페널티를 피하고, 정본 채널의 권위(도메인 신뢰도)를 쌓기 위한 규칙.

---

## 1. 정본(canonical) 지정

| 규칙 | 설명 |
|---|---|
| 1개 채널을 정본으로 지정 | 자사 도메인(홈페이지, 인블로그 등 커스텀 도메인 연결 채널) 우선. 그다음 브런치, 네이버 순 권장 |
| 정본 선택 기준 | (1) 자사 도메인 축적 효과가 있는가 (2) 테크니컬 SEO 통제권이 있는가 (3) 장기 자산으로 남는가 |
| 프로젝트별 정본 예시 | 위올라잇마이펫: 인블로그가 정본, Threads가 파생 / KDT: 자사 홈페이지 상세페이지가 정본 |

---

## 2. 변형 발행 원칙

| 규칙 | 설명 |
|---|---|
| 유사도 기준 | 정본 대비 **50% 미만 유사도**로 재작성 (전체 복사·붙여넣기 금지) |
| 유도 문구 | "전문은 [원본 링크]에서 확인하세요" 형태로 정본 링크 삽입 |
| 채널 특성 반영 | 단순 축약이 아니라 파생 채널의 포맷(숏폼, 캡션, 카드뉴스 등)에 맞게 재구성 |

---

## 3. 발행 시차

```
정본 먼저 발행 → 검색엔진 색인 확인(1~3일, 서치콘솔 URL 검사로 확인) → 파생 채널 발행
```

색인 전에 파생 콘텐츠를 먼저 노출시키면 검색엔진이 파생 채널을 원본으로 오인할 위험이 있다.

---

## 4. 백링크 설계

파생 콘텐츠 본문에 정본 URL을 삽입하면, 정본 도메인으로 유입 링크가 쌓여 도메인 권위(DA) 강화 효과가 있다. SNS(Threads, 인스타그램) 프로필/게시물의 링크도 UTM 파라미터를 붙여 유입 추적.

---

## 5. 채널별 정본-파생 관계 예시

| 정본 | 파생 | 파생 시 처리 |
|---|---|---|
| 인블로그/홈페이지 블로그 | Threads/인스타그램 | 핵심 메시지 3~5개 세트로 재구성, 유사도 50% 미만 |
| 인블로그/홈페이지 블로그 | 유튜브 스크립트 | 콘텐츠 구조는 유지하되 구어체로 재작성, 영상 설명란에 원문 링크 |
| 유튜브 영상 | 틱톡/숏폼 | 하이라이트 15~60초로 재편집, 캡션은 새로 작성(자동 자막 복붙 금지) |
| 상세페이지 | 마이크로사이트(이벤트) | 핵심 정보만 요약, 상세는 상세페이지로 링크 |
