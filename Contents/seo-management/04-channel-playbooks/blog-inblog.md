# 채널 플레이북 — 인블로그(Inblog)

> 공통 원칙(`../02-content-seo/writing-rules.md`)에 얹는 인블로그 전용 규칙. 출처: `content-project/SEOREADME.md` §3-3. 정본(canonical) 채널로 지정하기 가장 좋은 블로그형 플랫폼(자사 도메인 연결 가능 + 테크니컬 SEO 자동화 수준이 가장 높음).

| 항목 | 특성 및 규칙 |
|---|---|
| 주요 검색엔진 | 구글(Lighthouse SEO 만점 기록 플랫폼) + AI 검색엔진(AEO/GEO 대응) |
| 테크니컬 SEO | 자동 처리 — 사이트맵 자동 생성, 캐노니컬 URL, 301 리디렉트, SSL, JSON-LD 스키마, robots.txt, H1~H6 자동 구조화 |
| 도메인 | 커스텀 도메인 연결 필수(서브디렉토리 방식 권장: `example.com/blog`) — 자사 도메인 DA 축적에 유리 |
| 제목 | 타이틀 태그로 직접 반영. 60자 이내, E-E-A-T 기준 충족하는 전문성 있는 제목 |
| 메타 디스크립션 | 에디터에서 직접 입력. 120~150자, 키워드 + 가치 제안 |
| 본문 분량 | 2,000~4,000자(E-E-A-T 기준상 깊이 있는 콘텐츠 우대) |
| 헤딩 구조 | H1~H6 자동 태그 적용 → 마크다운 헤딩(`#`, `##`, `###`)을 정확히 사용해야 함 |
| 내부 링크 | 에디터에서 블로그 내 관련 글 연결 기능 지원 → 적극 활용 |
| CTA/리드폼 | CTA 버튼 삽입 + 하단 리드폼 배치 가능 → 전환 추적에 유리 |
| 분석 | 구글 서치콘솔 + GA 연동, 키워드별 유입/노출/게재순위 실시간 확인 |
| GEO/AEO | 높음 — FAQ 구조, 명확한 정의문, JSON-LD 활용 시 AI 검색엔진 답변 소스로 채택되기 유리 |
| 주의사항 | 노션 임베딩 시 원본 노션 페이지가 중복 노출되지 않도록 노션 측 SEO 설정 확인 |

## 발행 전 체크리스트
- [ ] 커스텀 도메인 연결 확인
- [ ] 구글 서치콘솔에 사이트맵 제출
- [ ] 메타 디스크립션 직접 입력(자동 생성 의존 금지)
- [ ] H1 → H2 → H3 계층 정확히 준수
- [ ] 내부 링크 1~3개 삽입
- [ ] CTA 버튼 또는 리드폼 배치
- [ ] JSON-LD 스키마 마크업 설정(해당 시) — `../01-technical-seo/structured-data-jsonld.md`
- [ ] AEO 요소(3줄 요약, FAQ, 정의문) 적용 — `../03-geo-aeo/geo-aeo-guide.md`
