# 채널 플레이북 — 인스타그램

> 검색엔진에 색인되지 않는 폐쇄형 플랫폼. "SEO"가 아니라 **플랫폼 자체 검색·탐색 알고리즘 최적화**로 치환해서 적용. `../00-framework.md` §2 판단 순서에 따라 01(테크니컬)은 축소 적용, 02(콘텐츠)는 플랫폼 검색·추천 최적화로 대체.

---

## 1. 플랫폼 자체 "테크니컬" 요소

| 항목 | 규칙 |
|---|---|
| 계정 정보(바이오) | 핵심 키워드 포함(예: "펫보험 정보" 같은 검색 가능 단어). 카테고리 태그 정확히 설정 |
| 프로필 링크 | bitly 등 UTM 파라미터 포함 링크 사용 — 유입 추적 |
| Alt 텍스트 | 게시물 업로드 시 이미지 대체 텍스트 직접 입력(자동 생성 대신) — 시각장애 접근성 + 검색 신호 |
| 위치 태그 | 로컬 비즈니스는 위치 태그로 지역 탐색 노출 |

## 2. 콘텐츠 "SEO" (탐색 탭 노출 최적화)

| 항목 | 규칙 |
|---|---|
| 캡션 키워드 | 첫 1~2문장에 핵심 키워드 자연 포함(인스타그램 검색이 캡션 텍스트를 색인) |
| 해시태그 | 콘텐츠와 직접 관련된 5~15개. 인기 태그 남용은 도달 저하 요인 |
| 커버 이미지/썸네일 | 릴스는 정지 화면에서도 메시지가 파악되는 커버 |
| 저장·공유 유도 | 알고리즘이 저장/공유를 좋아요보다 강한 신호로 취급 — 정보형 콘텐츠(체크리스트, 인포그래픽)가 유리 |
| 일관된 주제 | 계정 전체 주제 일관성이 탐색 탭 노출에 영향(네이버 C-Rank와 유사한 원리) |

## 3. 정본 연계

블로그/홈페이지 콘텐츠의 파생 콘텐츠로 운영 시 `../02-content-seo/cross-publishing.md` 원칙 적용 — 정본 링크를 캡션 또는 프로필에 배치, 유사도 50% 미만으로 재구성(단순 텍스트 축약이 아니라 카드뉴스/릴스 등 포맷 자체를 바꿀 것).
