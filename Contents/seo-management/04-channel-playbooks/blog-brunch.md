# 채널 플레이북 — 브런치(Brunch)

> 공통 원칙(`../02-content-seo/writing-rules.md`)에 얹는 브런치 전용 규칙. 출처: `content-project/SEOREADME.md` §3-2.

| 항목 | 특성 및 규칙 |
|---|---|
| 주요 검색엔진 | 구글(브런치 도메인 자체 DA가 높아 구글 노출에 유리) + 카카오 |
| 테크니컬 SEO | 브런치가 HTTPS·모바일 반응형·빠른 로딩을 기본 지원 → 자동 처리됨 |
| 제목 | 구글 타이틀 태그로 반영. 60자 이내 권장, 핵심 키워드 앞배치 |
| 부제목 | 메타 디스크립션 역할. 120자 이내, 키워드 + 클릭 유도 문구 |
| 본문 분량 | 2,000~3,500자 권장(브런치 독자는 긴 글에 익숙) |
| 소제목 | 브런치 에디터 '소제목'=H2, '중간 제목' 서식 활용 |
| 이미지 | 글 상단 대표 이미지 필수(SNS 공유 썸네일). 섹션 사이 이미지로 스크롤 리듬 |
| 키워드 태그 | 게시 시 '키워드' 3~5개(브런치 내부 검색 + 구글 태그 반영) |
| 매거진 기능 | 같은 주제 글을 매거진으로 묶으면 주제 관련성 강화(토픽 클러스터 효과) |
| 백링크 활용 | 브런치 글 URL을 타 채널에 공유 → 백링크 확보 → 구글 DA 상승 |
| GEO/AEO | 낮음 — 다만 구글 색인 대상이므로 기술 구조는 자동 처리됨 |
| 주의사항 | 브런치→네이버 동일 본문 복사 게시 시 중복 콘텐츠 페널티. 50% 미만 유사도로 변형 필요 |

## 발행 전 체크리스트
- [ ] 제목 60자 이내 + 부제목 120자 이내
- [ ] 대표 이미지 설정(SNS 공유 썸네일)
- [ ] 키워드 태그 3~5개 설정
- [ ] 관련 매거진에 글 포함
- [ ] 발행 후 외부 채널(링크드인/슬랙 등) 공유
