---

## 폴더 구조

파일은 **주제(카테고리)별 폴더**에 업로드하고, **파일명에 작성자 이니셜을 포함**하여 누가 올린 파일인지 구별합니다.

```
marketing-asset/
├── python/              # Python 자동화 스크립트
├── sql/                 # SQL 쿼리
├── json/                # 설정, 스키마, API 페이로드
├── prompts/             # AI 프롬프트 템플릿
├── contents/            # 콘텐츠 리소스 (텍스트, 마크다운 등)
└── README.md            # 이 파일
```

### 카테고리별 사용 가이드

| 폴더 | 저장 대상 | 파일 형식 |
|---|---|---|
| `python/` | 자동화 스크립트, 데이터 처리 코드 | `.py` |
| `sql/` | 조회 쿼리, 테이블 정의(DDL), 리포트용 쿼리 | `.sql` |
| `json/` | 환경 설정, 데이터 스키마, API 페이로드, 샘플 데이터 | `.json` |
| `prompts/` | LLM 시스템 프롬프트, 입력 템플릿, few-shot 예시 | `.md`, `.txt` |
| `contents/` | 마케팅 콘텐츠 초안, 템플릿, 발행 완료 원고 | `.md`, `.txt` |

## 파일명 규칙

같은 폴더에 여러 사람이 파일을 올리므로, **파일명에 작성자 이니셜을 반드시 포함**해야 합니다.

```
[이니셜]_[업무설명].[확장자]
```

**예시:**

| 파일명 | 설명 |
|---|---|
| `gjh_crm-sync.py` | 홍길동(gjh)이 만든 CRM 동기화 스크립트 |
| `mjk_monthly-report.sql` | 김민준(mjk)이 만든 월간 리포트 쿼리 |
| `gjh_ga4-event-schema.json` | 홍길동(gjh)이 만든 GA4 이벤트 스키마 |
| `mjk_blog-post-generator_v2.md` | 김민준(mjk)이 만든 블로그 생성 프롬프트 v2 |
| `gjh_march-newsletter-draft.md` | 홍길동(gjh)이 작성한 3월 뉴스레터 초안 |

### 파일명 작성 규칙 요약

| 항목 | 규칙 | 예시 |
|---|---|---|
| **이니셜** | 이름 영문 이니셜, 소문자 | `gjh` (홍길동), `mjk` (김민준) |
| **업무설명** | 영문 소문자, 단어는 하이픈(-) 구분 | `crm-sync`, `monthly-report` |
| **버전** | 업데이트 시 `_v2`, `_v3` suffix 추가 | `gjh_blog-prompt_v2.md` |
| **띄어쓰기** | 절대 사용 금지, 하이픈(-) 으로 대체 | `monthly-report` ✅  `monthly report` ❌ |

> 이니셜이 겹치는 경우 관리자에게 문의해 고유 이니셜을 지정받으세요.

## 처음 세팅하기 (최초 1회)

### 1단계: GitHub 계정 만들기

1. [github.com](https://github.com) 접속 → **Sign up**
2. 이메일, 비밀번호, 사용자명 입력 후 가입 완료
3. 관리자에게 **GitHub 사용자명** 전달 → 초대 링크 수신 → 수락

### 2단계: GitHub Desktop 설치 (권장)

명령어 없이 클릭만으로 사용할 수 있는 프로그램입니다.

1. [desktop.github.com](https://desktop.github.com) 에서 다운로드 및 설치
2. GitHub 계정으로 로그인
3. **File → Clone Repository** → `LIKELION-MKTDiv/marketing-asset` 선택
4. 내 PC에 저장할 위치 지정 → **Clone** 클릭

> 이제 내 PC에 `marketing-asset` 폴더가 생겼습니다. 이 폴더 안에서 작업하면 됩니다.

## 파일 올리는 방법

### 방법 A: GitHub Desktop 사용 (권장)

```
① 파일명에 본인 이니셜 포함 여부 확인 (예: gjh_crm-sync.py)
② 내 PC의 marketing-asset 폴더 내 해당 카테고리 폴더에 파일 추가/수정
③ GitHub Desktop 열기 → 변경된 파일 목록 자동 표시
④ 왼쪽 하단 Summary에 변경 내용 한줄 요약 입력
   예) "[python] gjh CRM 동기화 스크립트 v2 업로드"
⑤ [Commit to main] 버튼 클릭
⑥ 상단 [Push origin] 버튼 클릭 → 완료!
```

### 방법 B: GitHub 웹에서 직접 업로드

1. [이 저장소](https://github.com/LIKELION-MKTDiv/marketing-asset) 접속
2. 업로드할 카테고리 폴더로 이동 (예: `python/`, `sql/`)
3. **Add file → Upload files** 클릭
4. 파일명 확인 후 드래그 앤 드롭
5. 하단 "Commit changes"에 변경 내용 한줄 요약 입력
6. **Commit changes** 버튼 클릭 → 완료!

> **어떤 방법을 쓰든 결과는 동일합니다.** 편한 방법을 선택하세요.

## 커밋 메시지 작성 규칙

파일을 올릴 때 작성하는 한줄 요약(커밋 메시지)은 아래 형식을 따라 주세요.

```
[카테고리] 이니셜 작업내용 요약
```

**예시:**

| 커밋 메시지 | 설명 |
|---|---|
| `[python] gjh CRM 동기화 스크립트 최초 업로드` | 새 파일 추가 |
| `[sql] mjk 월간 리포트 쿼리 날짜 조건 수정` | 기존 파일 수정 |
| `[json] gjh GA4 이벤트 스키마 추가` | 새 파일 추가 |
| `[prompts] mjk 블로그 글 생성 프롬프트 v2 업데이트` | 버전 업데이트 |
| `[contents] gjh 3월 뉴스레터 초안 업로드` | 콘텐츠 추가 |

## 주의사항

### ⛔ 절대 올리면 안 되는 것

- **API 키, 비밀번호, 토큰** 등 인증 정보
- **고객 개인정보**가 포함된 데이터 파일
- `.env` 파일 (환경변수 설정 파일)

> 민감 정보는 코드에 직접 쓰지 말고 환경변수로 분리하세요.  
> JSON 파일 내 API 키 값은 `"YOUR_API_KEY"` 형태의 placeholder로 대체 후 업로드하세요.  
> 실수로 올렸다면 즉시 관리자에게 알려주세요.

### ⚠️ 작업 전 습관

- **작업 시작 전**: GitHub Desktop에서 **Fetch origin** → **Pull** 클릭 (최신 상태로 동기화)
- **작업 완료 후**: Commit → Push (내 작업을 올리기)

## 자주 묻는 질문

**Q. 같은 파일을 두 사람이 동시에 수정하면 어떻게 되나요?**
A. GitHub가 "충돌(Conflict)"이 발생했다고 알려줍니다. 당황하지 마시고 관리자에게 알려주세요. 함께 해결할 수 있습니다.

**Q. 실수로 파일을 삭제하거나 잘못 수정했어요.**
A. GitHub에는 모든 변경 이력이 남아 있으므로 이전 버전으로 되돌릴 수 있습니다. 관리자에게 요청해 주세요.

**Q. 프롬프트나 콘텐츠 같은 텍스트 파일은 어떤 형식으로 올리나요?**
A. `.md`(마크다운) 또는 `.txt` 파일로 저장해서 올려주세요. GitHub에서 바로 미리보기가 가능합니다.

**Q. 이니셜이 다른 팀원과 겹쳐요.**
A. 관리자에게 문의해 고유 이니셜을 별도로 지정받으세요. 이니셜이 확정된 후 파일을 업로드해 주세요.

**Q. 용량이 큰 파일(영상, 대용량 데이터)도 올릴 수 있나요?**
A. GitHub는 파일당 100MB 제한이 있습니다. 대용량 파일은 Google Drive 링크를 README나 `.md` 파일에 기록하는 방식을 권장합니다.
