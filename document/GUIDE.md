## 1. 저장소 개요

이 저장소는 마케팅본부에서 사용하는 **자동화 스크립트, 데이터 처리 코드, 프롬프트 템플릿, 콘텐츠 자산**을 한곳에 모아 팀 전체가 공유하고 재사용할 수 있도록 관리합니다.

| 목적 | 설명 |
|------|------|
| 코드 백업 | 개인 PC에만 있는 스크립트를 팀 전체가 접근 가능한 공간에 보관 |
| 버전 관리 | 파일 수정 이력이 자동으로 기록되어 이전 버전 복구 가능 |
| 인수인계 | 담당자가 바뀌어도 동일한 기준으로 운영 가능 |
| 협업 | 팀원 간 코드 공유 및 재활용 |

## 2. 폴더 구조

```
marketing-asset/
├── AppsScript/     # Google Apps Script (.gs, .gs.txt)
│   └── TIL/        # TIL 챌린지 검수 관련 스크립트
├── Python/         # Python 자동화 스크립트 (.py, .ipynb)
├── SQL/            # SQL 쿼리 (.sql)
├── json/           # n8n 워크플로우, 설정, API 페이로드 (.json)
├── Prompt/         # AI 프롬프트 템플릿 (.md, .txt)
├── Contents/       # 콘텐츠 리소스 (마크다운, 텍스트)
├── document/       # 팀 내부 문서 및 가이드
└── GUIDE.md        # 이 파일
```

### 카테고리별 사용 기준

| 폴더 | 저장 대상 | 파일 형식 |
|------|-----------|-----------|
| `AppsScript/` | Google Sheets 연동 자동화 스크립트 | `.gs`, `.gs.txt` |
| `Python/` | 자동화 스크립트, 데이터 처리, 분석 | `.py`, `.ipynb` |
| `SQL/` | 조회 쿼리, 테이블 정의(DDL), 리포트 쿼리 | `.sql` |
| `json/` | n8n 워크플로우, 환경 설정, API 페이로드 | `.json` |
| `Prompt/` | LLM 시스템 프롬프트, 입력 템플릿, few-shot 예시 | `.md`, `.txt` |
| `Contents/` | 마케팅 콘텐츠 초안, 템플릿, 발행 원고 | `.md`, `.txt` |

## 3. 파일명 규칙

같은 폴더에 여러 사람이 파일을 올리므로, **파일명에 반드시 본인 이니셜을 포함**해야 합니다.

```
[이니셜]_[업무설명].[확장자]
```

### 예시

| 파일명 | 설명 |
|--------|------|
| `gjh_crm-sync.py` | 홍길동(gjh)이 만든 CRM 동기화 스크립트 |
| `mjk_monthly-report.sql` | 김민준(mjk)이 만든 월간 리포트 쿼리 |
| `gjh_ga4-event-schema.json` | 홍길동(gjh)이 만든 GA4 이벤트 스키마 |
| `mjk_blog-post-generator_v2.md` | 김민준(mjk)이 만든 블로그 생성 프롬프트 v2 |

### 파일명 작성 규칙 요약

| 항목 | 규칙 | 예시 |
|------|------|------|
| **이니셜** | 이름 영문 이니셜, 소문자 | `gjh` (홍길동), `mjk` (김민준) |
| **업무설명** | 영문 소문자, 단어는 하이픈(-) 구분 | `crm-sync`, `monthly-report` |
| **버전** | 업데이트 시 `_v2`, `_v3` suffix 추가 | `gjh_blog-prompt_v2.md` |
| **띄어쓰기** | 절대 사용 금지, 하이픈(-)으로 대체 | `monthly-report` ✅ `monthly report` ❌ |

> 이니셜이 다른 팀원과 겹치는 경우 관리자에게 문의해 고유 이니셜을 지정받으세요.

## 4. 처음 세팅하기

### 1단계: GitHub 계정 만들기

1. [github.com](https://github.com) 접속 → **Sign up**
2. 이메일, 비밀번호, 사용자명 입력 후 가입 완료
3. 관리자에게 **GitHub 사용자명** 전달 → 초대 링크 수신 → 수락

### 2단계: GitHub Desktop 설치 (권장)

명령어 없이 클릭만으로 사용할 수 있는 GUI 프로그램입니다.

1. [desktop.github.com](https://desktop.github.com) 에서 다운로드 및 설치
2. GitHub 계정으로 로그인
3. **File → Clone Repository** → `LIKELION-MKTDiv/marketing-asset` 선택
4. 내 PC에 저장할 위치 지정 → **Clone** 클릭

이제 내 PC에 `marketing-asset` 폴더가 생겼습니다. 이 폴더 안에서 작업하면 됩니다.

## 5. 파일 올리는 방법

### 방법 A: GitHub Desktop 사용 (권장)

```
① 파일명에 본인 이니셜 포함 여부 확인 (예: gjh_crm-sync.py)
② 내 PC의 marketing-asset 폴더 내 해당 카테고리 폴더에 파일 추가 또는 수정
③ GitHub Desktop 열기 → 변경된 파일 목록이 자동으로 표시됨
④ 왼쪽 하단 Summary에 변경 내용 한줄 요약 입력
   예) "[AppsScript] gjh TIL 검수 스크립트 v2 업로드"
⑤ [Commit to main] 버튼 클릭
⑥ 상단 [Push origin] 버튼 클릭 → 완료!
```

### 방법 B: GitHub 웹에서 직접 업로드

1. [이 저장소](https://github.com/LIKELION-MKTDiv/marketing-asset) 접속
2. 업로드할 카테고리 폴더로 이동 (예: `AppsScript/`, `Python/`)
3. **Add file → Upload files** 클릭
4. 파일명 확인 후 드래그 앤 드롭
5. 하단 "Commit changes"에 변경 내용 한줄 요약 입력
6. **Commit changes** 버튼 클릭 → 완료!

> 어떤 방법을 쓰든 결과는 동일합니다. 편한 방법을 선택하세요.

### 작업 전 필수 습관

- **작업 시작 전**: GitHub Desktop에서 **Fetch origin → Pull** 클릭 (최신 상태 동기화)
- **작업 완료 후**: Commit → Push (내 작업 업로드)

### 7.1 TIL 블로그 키워드 검수 스크립트

**파일 위치:** `AppsScript/TIL/ksy_til-keyword-validation.gs`
**상세 가이드:** `AppsScript/TIL/TIL_제목_해시태그_분류_Apps_Script_활용_가이드.md`

#### 목적

TIL 챌린지 참여자가 제출한 블로그 URL을 자동으로 검수하여 경품 지급 대상자를 판별합니다.
기존에는 월 300건 이상의 URL을 수동으로 확인하는 데 2~3시간이 소요되었으나, 이 스크립트로 **10분 이내**로 단축되었습니다.

#### 동작 원리

1. 스프레드시트 **A열**의 블로그 URL을 순서대로 읽음
2. 각 URL에 접속하여 HTML을 분석
3. 플랫폼별 제목 및 해시태그 영역 추출
4. `멋쟁이사자처럼` 키워드 포함 여부 확인
5. 결과를 스프레드시트에 자동 입력

#### 스프레드시트 열 구조

| 열 | 입력 방식 | 내용 |
|----|-----------|------|
| A | 수동 입력 | 블로그 URL |
| F | 자동 입력 | 제목에 키워드 포함 시 `O`, 미포함 시 `X` |
| G | 자동 입력 | 해시태그에 키워드 포함 시 `O`, 미포함 시 `X` |
| H | 자동 입력 | F, G 모두 `O`이면 `O`, 하나라도 `X`이면 `X` |
| I | 자동 입력 | 네이버 태그 자동추출 실패 시 `태그 자동추출 불가` 로그 |

#### 실행 방법

1. Google Spreadsheet 열기 (A열에 URL 입력 완료 상태)
2. 상단 메뉴 **확장 프로그램 → Apps Script** 클릭
3. 스크립트 편집기에서 `checkBlogLinks` 함수 선택
4. **▶ 실행** 버튼 클릭
5. F/G/H열 결과 자동 반영 확인

#### 플랫폼별 주의사항

| 플랫폼 | 자동 검수 | 주의사항 |
|--------|-----------|----------|
| **네이버 블로그** | 제목 O / 태그 △ | I열에 `태그 자동추출 불가` 표시 시 G열 **수동 확인 필요** |
| **벨로그 (Velog)** | 제목 O / 태그 O | 자동 검수 정확도 높음, 별도 보완 불필요 |
| **티스토리** | 제목 O / 태그 △ | 스킨(테마)에 따라 오탐 가능. 오류 발생 시 관리자에게 문의 |
| **노션** | 제목 O / 태그 X | 노션은 태그 구조 없음. G열 전체 X → 운영 정책별 별도 기준 적용 |

> ⚠️ 네이버 블로그는 자동 검수 결과 확인 후, `태그 자동추출 불가` 건은 반드시 수동으로 보완하세요.

### 7.2 부트캠프 데이터 동기화 스크립트

**파일 위치:** `AppsScript/bootcamp_data_sync_script.gs.txt`

#### 목적

RAW 데이터 폴더에 있는 부트캠프 프로젝트별 지원자 데이터를 스프레드시트에 자동으로 업데이트·적재하고, 최초작성일 기준으로 자동 정렬합니다.

#### 동작 원리

1. 현재 스프레드시트 파일명에서 프로젝트 코드(`kdt-xxxxx`) 자동 추출
2. Google Drive RAW 폴더에서 해당 프로젝트 코드가 포함된 최신 파일 검색
3. 파일 데이터 로드 (CSV 및 Google Sheets 형식 모두 지원)
4. `user_id + 최초작성일` 복합 키로 기존 데이터와 비교
   - 이미 있는 데이터 → 업데이트
   - 신규 데이터 → 추가
5. 최초작성일 기준 오름차순 자동 정렬
6. 완료 후 처리 결과 알림창 표시

#### 실행 방법

1. 프로젝트 스프레드시트 열기 (파일명에 `kdt-` 프로젝트 코드 포함 필수)
2. **확장 프로그램 → Apps Script** 클릭
3. `updateBootcampData` 함수 선택 후 **▶ 실행**
4. 알림창에서 업데이트/신규 추가 건수 확인

#### 사전 조건

| 항목 | 내용 |
|------|------|
| 스프레드시트 파일명 | `kdt-[프로젝트코드]` 형식 포함 필수 |
| RAW 폴더 ID | 스크립트 내 `FOLDER_ID` 상수 확인 (변경 시 관리자 문의) |
| RAW 파일명 | 동일한 프로젝트 코드 포함 필수 |
| 필수 컬럼 | `user_id`, `최초작성일` 두 컬럼이 반드시 존재해야 함 |

### 7.3 n8n IT 트렌드 콘텐츠 자동화 워크플로우

**파일 위치:** `json/IT Development Trend Research and Multi-Format Content Creation (1).json`

#### 목적

매일 오전 9시, IT 개발 트렌드를 자동으로 수집하고 다양한 형식의 콘텐츠(블로그, 뉴스레터 등)를 생성하는 n8n 자동화 워크플로우입니다.

#### 워크플로우 구성

| 노드 | 역할 |
|------|------|
| Schedule Trigger | 매일 오전 9시 자동 실행 |
| Workflow Configuration | 날짜, 타겟 오디언스 등 기본 설정 주입 |
| Read Tech News RSS Feeds | Hacker News RSS에서 최신 기술 뉴스 수집 |
| GitHub Trending 수집 | 최근 7일 인기 오픈소스 저장소 수집 |
| Claude AI 분석 | 수집된 트렌드를 분석하여 콘텐츠 생성 |

#### n8n에 임포트하는 방법

1. n8n 대시보드 접속
2. 우측 상단 **Import from file** 클릭
3. `json/IT Development Trend Research and Multi-Format Content Creation (1).json` 파일 선택
4. 워크플로우 불러오기 완료
5. API 키 등 환경변수 설정 후 활성화

> 워크플로우에 사용되는 API 키는 코드에 직접 넣지 말고 n8n의 Credentials 기능을 활용하세요.

## 9. 자주 묻는 질문

**Q. 같은 파일을 두 사람이 동시에 수정하면 어떻게 되나요?**
A. GitHub가 "충돌(Conflict)"이 발생했다고 알려줍니다. 당황하지 마시고 관리자에게 알려주세요.

**Q. 실수로 파일을 삭제하거나 잘못 수정했어요.**
A. GitHub에는 모든 변경 이력이 남아 있으므로 이전 버전으로 되돌릴 수 있습니다. 관리자에게 요청해 주세요.

**Q. 프롬프트나 콘텐츠 같은 텍스트 파일은 어떤 형식으로 올리나요?**
A. `.md`(마크다운) 또는 `.txt` 파일로 저장해서 올려주세요. GitHub에서 바로 미리보기가 가능합니다.

**Q. 이니셜이 다른 팀원과 겹쳐요.**
A. 관리자에게 문의해 고유 이니셜을 별도로 지정받으세요.

**Q. 용량이 큰 파일(영상, 대용량 데이터)도 올릴 수 있나요?**
A. GitHub는 파일당 100MB 제한이 있습니다. 대용량 파일은 Google Drive 링크를 README나 `.md` 파일에 기록하는 방식을 권장합니다.

**Q. Apps Script를 처음 실행하면 권한 요청 팝업이 뜨는데요?**
A. Google 계정에 스크립트 실행 권한을 부여하는 과정입니다. `고급 → [프로젝트명](으)로 이동 → 허용` 순서로 진행하면 됩니다.

**Q. n8n 워크플로우를 처음 임포트했는데 실행이 안 돼요.**
A. API 키 등 Credentials 설정이 필요합니다. 워크플로우 내 빨간 표시 노드를 클릭해 누락된 설정을 확인하거나 관리자에게 문의하세요.
