## 0. 고객 여정 (Customer Journey)

| 단계 | 터치포인트 | 채널톡 연결 |
|------|---------|-----------|
| **1. 인지·유입** | 광고·SNS·검색 → 홈페이지 접속 | 마케팅 캠페인 팝업 노출 |
| **2. 탐색·관심** | 상세 페이지 확인 → 상담창(라운지) 오픈 | 서포트봇 먼저 말 걸기 ("어떤 과정이 궁금하신가요?") |
| **3. 질문·상담** | ALF 답변 또는 매니저 연결 | 관심 강좌 태그 자동 부여 → 담당 팀 배정 |
| **4. 의사결정·전환** | 답변 확인 후 수강신청·결제 | ALF 도큐먼트 기반 답변 또는 매니저 상세 안내 |
| **5. 수강·관리** | 캠프 시작 후 문의 재인입 | 기존 상담 이력 기반 연속성 있는 케어 |

## 1. 전체 플로우

```
멋쟁이사자처럼 홈페이지 접속
      ↓
우측 하단 채팅 아이콘 클릭
      ↓
[문의하기] 버튼 클릭
      ↓
ALF 워크플로우 진입
      ↓
[1단계] 문의 인입 — 고객 메시지 전송 / 서포트봇 종료 시 ALF 분석 시작
      ↓
[2단계] 의도 분석 — 질문 의도 파악 + 고객 정보·대화 이력 실시간 확인
      ↓
[3단계] 지식 검색 — FAQ·아티클 검색(search_document)
      ↓
[4단계] 답변 생성 — 출처(Citation) 기반 답변 생성 + 브랜드 톤앤매너 적용
      ↓
[5단계] 응대 모드 분기
    ┌──────────────────────────────────────┐
 CASE A                               CASE B
 고객 ALF (자동 응대)             팀 ALF (매니저 어시스턴트)
 정답 확률 높음 → 즉시 전송       내부 창에 초안 제공
 고객 추가 반응 모니터링          매니저 확인·수정 후 전송
    │                                  │
    ↓                                  ↓
ALF 해결                       매니저 직접 응대
    ↓                                  ↓
 종료                     ┌────────────────────┐
                       즉시 해결       외부 확인 필요
                          ↓                ↓
                        종료      고용24 / 카드사 등
                                   별도 직접 확인
                                   (채널톡 내 조회 불가)
                                          ↓
                                        종료
      ↓
[6단계] 사후 처리 — 상담 요약 생성 / 매니저 피드백 → ALF 학습
```

## 2. 설정된 워크플로우 구성

| 단계 | 내용 |
|------|------|
| **1. 트리거** | 상담창 열림 시 즉시 시작 / AINLP·클라우드엔지니어링 상세 페이지 방문 시 전용 워크플로우 작동 |
| **2. 자동 정보 수집** | 버튼형 메뉴로 관심 과정 선택 유도 / 이름·연락처 등 개인정보 수집 후 고객 프로필 자동 업데이트 |
| **3. 담당자·팀 배정** | 기술 문의 → 교육팀 / 결제 관련 → 운영팀 자동 배정 / **상담 태그 → 슬랙 전송 → 태그별 팀 멘션 자동 발송** / 미응답 시 재멘션으로 답변 속도 보완 (수동) |
| **4. ALF 답변 보조** | 지식베이스 연동 → 등록된 강좌 정보 기반 답변 초안 생성 또는 고객 직접 응대 |
| **5. 상담 종료 및 후속** | 일정 시간 무응답 또는 해결 시 자동 종료 / '수강신청', '상담예약' 등 태그 자동 부여 |

## 3. 상담 태그 매칭 구조

상담 태그는 아래 세 가지가 결합되어 자동 매칭됩니다.

| 방식 | 설명 |
|------|------|
| **키워드 매칭** | 고객 발화의 핵심 키워드 분석 → 직접 일치 또는 문맥 기반 의도 파악 |
| **서포트봇·워크플로우** | 고객이 선택한 버튼(예: [환불 문의]) 또는 진입 경로(페이지·캠페인)에 따라 시작부터 자동 태그 부여 |
| **ALF 추론** | 상담 전체 맥락 요약 후 관련 태그 추천 / 매니저 수동 태그 이력 학습 → 정확도 향상 |

> 매니저가 태그를 수동 보정하면 해당 이력이 학습 데이터로 활용되어 이후 유사 문의의 자동 매칭 정확도가 높아집니다.

### ⚠️ 태그 오매칭 발생 원인

| 원인 | 상황 | 문제 |
|------|------|------|
| **유입 경로 자동화 규칙** | 결제 페이지에서 상담 시작 → '결제문의' 태그 자동 부여 | 실제 문의 내용과 무관하게 위치 기반 태그가 우선 적용됨 |
| **세션 기록 우선** | 여러 페이지 이동 후 메시지 전송 | 상담창이 처음 열렸던 페이지의 태그 규칙이 적용됨 |
| **서포트봇 고정 태그** | [이벤트 문의] 버튼 선택 후 "환불하고 싶어요"로 변심 | 처음 선택한 버튼의 태그가 그대로 유지됨 |

> **영향:** 태그 오매칭 → 분류 데이터 왜곡 → VOC 분석 정확도 저하

### 🔧 현재 대응 방식 (수동)

채널톡 상담 태그와 슬랙 멘션이 불일치할 경우, **슬랙 #channeltalk-qna 채널에서 담당자를 직접 수동 멘션**하여 빠른 응대를 요청하는 방식으로 운영 중입니다.

> **한계:** 태그 오매칭 건마다 수동 확인·수정이 필요 → 운영 리소스 소모 / 누락 위험 존재

### ⚠️ 추가 개선 필요 사항

**담당자의 이전 상담 내용 미확인 문제**

매니저가 배정된 후 이전 대화 이력을 확인하지 않고 바로 답변하는 사례가 발생하고 있습니다.

- 고객이 이미 설명한 내용을 다시 설명해야 하는 상황 발생
- 중복 질문으로 고객 불만·이탈 위험 증가
- ALF가 생성한 상담 요약(6단계)을 매니저가 활용하지 않는 구조적 문제

> **개선 방향:** 매니저 배정 시 ALF 요약 및 이전 대화 이력을 필수 확인하도록 프로세스 정비 필요

## 4. 병목 발생 지점

### 🔴 병목 1 — ALF 미매칭으로 인한 상담원 연결 과다

- 전체 분석 대상 65건 중 **37건(56.9%)** 이 상담원 연결 요청으로 이어짐
- 주요 원인: 국취제·내일배움카드 등 복잡한 정책 문의가 ALF 지식베이스에 미등록
- **결과:** 봇이 처리 가능한 문의도 매니저로 넘어가 응대 리소스 낭비

### 🔴 병목 2 — 매니저 배정 후 응답 지연

- 과정별 담당 매니저가 명확히 지정되지 않아 배정 후에도 응답이 늦어지는 구조
- **KDT/CLD 평균 응대 시간 16분 24초** (최대 1시간 14분 발생)
- KDC·AIERS 등 단순 문의는 1~2분 내외 처리 → 과정별 편차 큼

### 🔴 병목 3 — 외부 시스템 확인 필요 건

- 고용24 연동 오류, 내일배움카드 결제 오류(F0004), 카드사 한도 문제 등
- 채널톡 내에서 외부 시스템(고용24·카드사) 조회 기능이 없어 매니저가 **직접 별도로 확인**해야 함
- 이로 인해 **평균 13~24분** 소요 — 채널톡 구조적 한계
