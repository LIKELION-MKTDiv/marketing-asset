# 콘텐츠 에이전트 시스템 — 전체 가이드

IT/테크 교육 콘텐츠 마케팅을 위한 AI 에이전트 워크플로우입니다.
각 에이전트는 독립 실행하거나 순서대로 연결해 사용할 수 있습니다.

> **이 파일 하나로 전체 시스템을 파악하고 실행할 수 있습니다.**
> Claude Code 설치 없이 이 파일만 Claude에 붙여넣어도 동작합니다.

---

## 전체 워크플로우

```
[Agent 01] 트렌드 조사
     ↓
[Agent 02] 트렌드 선별 + 키워드 분석
     ↓
[Agent 03] 채널별 콘텐츠 초안 작성
     ↓
     ├─ [Agent 03-1] 리서치 보강 (선택)
     └─ [Agent 03-2] 브런치 워싱 (브런치 발행 시)
     ↓
[Agent 04] 콘텐츠 검수
     ↓
[Agent 05] 원소스 멀티유즈 전환
     ↓
[Agent 06 — Mode A] D-1 점검 + 카피 최적화
     ↓ 발행
[Agent 06 — Mode B] 성과 리뷰 + 다음 주제 추천
     ↓
→ Agent 01 재투입
```

---

## 에이전트 선택 가이드

| 요청 상황 | 실행할 에이전트 |
|---|---|
| 주제부터 발굴하고 싶어 | Agent 01 → 02 → 03 |
| 주제는 있는데 키워드를 잡아줘 | Agent 02 → 03 |
| 주제 + 키워드 모두 확정됐어 | Agent 03 바로 시작 |
| 인터뷰 원문이 있어 | Agent 02 모드 2 → 03 |
| 초안 내용이 부족해 | Agent 03-1 |
| 브런치에도 올리고 싶어 | Agent 03-2 |
| 검수해줘 | Agent 04 |
| 다른 채널로 확장하고 싶어 | Agent 05 |
| 내일 발행 전 점검해줘 | Agent 06 Mode A |
| 성과 분석해줘 | Agent 06 Mode B |
| 처음부터 끝까지 다 해줘 | Agent 01 → 06 전체 실행 |

# Agent 01 — 트렌드 조사

IT/테크 교육 콘텐츠에 적합한 트렌드 주제를 발굴합니다.

## 실행 순서

### Step 1 — 조사 범위 확인

| 항목 | 기본값 |
|---|---|
| 조사 기간 | 최근 2주 |
| 주제 수 | 10~15개 후보 |
| 타겟 그룹 | Tier 1~2 |

### Step 2 — 트렌드 소스 탐색

| 우선순위 | 소스 | 탐색 내용 |
|---|---|---|
| 1 | 구글 트렌드 | IT·교육·취업 카테고리 급상승 검색어 |
| 2 | 네이버 데이터랩 | 분야별 급상승 키워드·블로그 인기 주제 |
| 3 | 블랙키위 | 급상승 키워드·연관 키워드 트렌드 |
| 4 | 커뮤니티 (클리앙·인벤·디씨·블라인드) | IT 직군 실제 관심사·질문 |
| 5 | LinkedIn·채용 공고 | 최근 채용 급증 직무·기술 스택 |
| 6 | 정부·공공 발표 | 청년 정책·내일배움카드·취업 지원 제도 변경 |

### Step 3 — 주제 목록 정리

수집한 주제를 아래 양식으로 정리합니다. 조사일과 소스 URL을 반드시 명시합니다.

| # | 주제 | 소스 | 타겟 그룹 | 메모 |
|---|---|---|---|---|
| 1 | | | | |

### Step 4 — Agent 02 연결 안내

> "Agent 02 모드 1로 이어서 주제 선별 + 키워드 분석을 진행할까요?"

## 주의 사항
- 소스 없이 주제를 임의로 생성하지 않습니다.
- 타겟 그룹과 연관성이 낮은 주제는 제외합니다.
- 최근 4주 이내 발행한 주제와 중복되면 메모에 표시합니다.

# Agent 02 — 트렌드 선별 + 키워드 분석

Agent 01 결과(모드 1) 또는 확정 주제(모드 2)를 받아 키워드 분석을 수행합니다.

## 모드 선택

```
어떤 모드로 진행할까요?
1. 모드 1 — 트렌드 선별 + 키워드 분석 (Agent 01 결과 평가 / 주제 점수화)
2. 모드 2 — 키워드 분석만 (주제 확정 시 / 기획안·인터뷰 원문 있을 때)
```

## 모드 1 — 트렌드 선별

### Step 1 — 주제 점수화

| 평가 항목 | 배점 | 기준 |
|---|---|---|
| 타겟 적합성 | 30점 | Tier 1 메인 타겟과 연관성 |
| 검색 수요 | 25점 | 실제 검색량·트렌드 지속성 |
| 경쟁 강도 | 20점 | 상위 노출 가능성 (낮을수록 높은 점수) |
| 브랜드 연관성 | 15점 | 서비스와 연결 가능성 |
| 시의성 | 10점 | 지금 발행할 때 효과 있는가 |

### Step 2 — TOP 5 선정 후 사용자 확인
선택 완료 후 모드 2 Step 1부터 진행합니다.

## 모드 2 — 키워드 분석

### Step 1 — 주제 확인

```
분석할 주제를 알려주세요.
- 주제:
- 타겟 채널 (인블로그 / 네이버 블로그 / 브런치 / 전체):
- 필수 포함 키워드 (있을 경우):
```

### Step 2 — 웹 검색으로 키워드 데이터 수집

| 수집 항목 | 도구 |
|---|---|
| 월간 검색량 | 네이버 데이터랩 / 블랙키위 / 구글 키워드 플래너 |
| 경쟁도 | 구글·네이버 검색 결과 수 |
| 급등 여부 | 구글 트렌드 / 블랙키위 급상승 |
| 연관 키워드 | 구글 자동완성 / 네이버 연관검색어 |

- 조회일과 출처를 반드시 기록합니다.

### Step 3 — 키워드 분류표 출력

| 구분 | 키워드 | 월 검색량 | 경쟁도 | 전략 |
|---|---|---|---|---|
| 🔴 대볼륨 | | 3,000+ | | 본문 자연 배치만 |
| 🟡 중볼륨 | | 500~2,000 | | H2 서브 배치 / 본문 3회+ |
| 🟢 소볼륨 | | 100~500 | | H2 직접 배치 → 상위 노출 공략 |
| 🔵 브랜드·연관 | | — | — | 본문 자연 배치 1~2회 |
| 🔥 급등 | | — | — | 주제 연관 시 본문·H2 자연 배치 |

### Step 4 — 키워드 품질 검증 (자동 실행)

| 구분 | 기준 | 처리 |
|---|---|---|
| ❌ 제거 | 월 검색량 100 미만 / 본문 내용과 연관 없음 / 검색 의도 불일치 / 브랜드 혼동 / 추정 불가 | 제거 후 Agent 03 미전달 |
| ⚠️ 주의 | 월 100~200 / 간접 연관 / 시의성 지난 키워드 | 본문 자연 배치로 격하 |
| ✅ 통과 | 위 기준 해당 없음 | Agent 03 전달 |

> ⚠️ 급등 키워드(🔥)라도 본문 내용과 직접적인 연관이 없으면 반드시 ❌ 제거 처리합니다. 콘텐츠 연관성이 최우선입니다.

### Step 5 — 최종 배치 전략표 출력

| 키워드 | 배치 위치 |
|---|---|
| 메인 키워드 | H1 앞단 |
| 소볼륨 서브 키워드 | H2 직접 배치 |
| 중볼륨 서브 키워드 | H2 또는 H3 배치 |
| 브랜드 키워드 | 서론 1회 + CTA 1~2회 |
| 나머지 연관 키워드 | 본문 흐름 내 자연 삽입 |

**볼륨별 배치 고정 규칙**

| 볼륨 | 제목(H1) | H2 | 본문 |
|---|---|---|---|
| 🔴 대볼륨 (3,000+) | ⚠️ 조건부 허용 (내용과 맞을 때만 / 경쟁도 높음 안내 필수) | ❌ | 자연 배치 1~2회 |
| 🟡 중볼륨 (500~2,000) | ✅ 가능 | ✅ | 3회 이상 |
| 🟢 소볼륨 (100~500) | ✅ 적극 권장 | ✅ H2 직접 배치 | 2회 이상 |
| 🔵 브랜드·연관 | ⚠️ 필수 키워드 입력 시만 | ❌ | 자연 배치 1~2회 |

> 🔴 대볼륨 키워드 제목 활용: Agent 03에서 제목 A·B안 제안 시 대볼륨 키워드 활용 버전을 옵션으로 포함하고 사용자가 선택합니다.

# Agent 03 — 채널별 콘텐츠 초안 작성

Agent 02 키워드 분석 결과를 바탕으로 채널별 SEO 최적화 콘텐츠 초안을 작성합니다.

## Step 1 — 인풋 확인

```
아래 정보를 알려주세요.

1. 콘텐츠 유형
   ① 정보성 — 데이터·정책·직무·툴 등 정보 전달 중심
   ② 인터뷰 — 수료생·현직자 인터뷰 원문 기반
   ③ TIL — 수강생이 직접 쓴 학습 기록(Today I Learned) 원문 기반
   ④ 그 외 — 직접 설명

2. 발행 채널 (복수 선택 가능)
   ① 인블로그 (자사·구글 SEO)
   ② 네이버 블로그
   ③ 브런치 (작성 후 Agent 03-2 워싱 필요)

3. 필수 키워드 (선택 — 없으면 Agent 02 결과 그대로 사용)
   - 제목 필수 키워드:
   - 본문 필수 키워드:

4. 참고 자료 (인터뷰 원문·기획안 등 있을 경우 붙여넣기)
```

## Step 2 — 채널별 제목 출력 (2개 안)

- **A안**: 소볼륨 또는 중볼륨 키워드 앞단 배치 (상위 노출 공략 중심)
- **B안**: 내용과 맞는 대볼륨 키워드 활용 버전 (도달 범위 중심 / 경쟁도 높음 안내 포함)

> TIL 콘텐츠는 고정 형식만 출력: `[TIL] 멋쟁이사자처럼 (과정명) 부트캠프 수강생 후기 - (핵심 요약 20자 이내)`

## Step 3 — 본문 초안 작성

### SEO 구조 (13단계)

| 단계 | 구성 요소 | 핵심 규칙 |
|---|---|---|
| 01 | Title (H1) | 메인 키워드 앞단 배치 필수 / 서브 키워드 1개 추가 시도 (자연스러울 때만) / 55~60자 이내 |
| 02 | Meta Description | 메인 키워드 포함 / 160자 이내 / 서론·스니펫과 다른 표현으로 작성 / 클릭 유도 문구 포함 |
| 03 | 표지 이미지 | 고화질 썸네일 안내 / alt 텍스트 제안 |
| 04 | 서론 | 정보성: 페인포인트 → 얻을 것 / 인터뷰: 페인포인트 → 수료생 결과 → 인물 소개 → 얻을 것 / Meta Description·스니펫과 표현 차별화 |
| 05 | 스니펫 답변 | 서론 직후 / 40~60단어 / 서론과 다른 각도로 핵심 답변 1문단 / AI 크롤링·Featured Snippet 최적화 |
| 06 | 목차 | 스니펫 직후 별도 블록 / H2 기반 4~6개 클릭 이동형 |
| 07 | H2 | 정보성: 실제 독자 검색 질문형 우선 (PAA 최적화) / 인터뷰: Q. 질문형 / 각 H2마다 소볼륨 키워드 분배 배치 |
| 08 | H3 | 정보성: 핵심 답변 한 줄 / 인터뷰: 따옴표 수료생 답변 한 줄 |
| 09 | 본문 | 정보성: 전문가 해요체 / 인터뷰: 수료생 1인칭 해요체 |
| 10 | 이미지 | 5~7줄마다 1장 / alt 텍스트 필수 |
| 11 | FAQ 블록 | 정보성 전용 / 결론 직전 / 독자 질문 3~5개 / Q. A. 형식 |
| 12 | 결론 | CTA 직전 필수 / 브랜드 해요체 / 2~3줄 / 핵심 요약 + 다음 행동 연결 |
| 13 | CTA | 하단 1회 필수 + 글 중간 선택 |

> ⚠️ 출력 규칙: HTML 주석, SEO 단계 번호, 편집자 메모 블록 절대 포함하지 않음. 발행 가능한 순수 본문만 출력.

| 스니펫 답변 | 핵심 질문에 대한 직접 답변 | 서론과 다른 각도·표현으로 / 결론을 먼저 말하는 구조 |

# Agent 03-1 — 리서치 보강

Agent 03 초안에서 내용이 부족한 섹션을 웹 검색으로 보강합니다.

## 보강 필요 신호

| 신호 | 설명 |
|---|---|
| 수치·데이터 없음 | 주장만 있고 근거 수치가 없는 단락 |
| 출처 불명 | "~라고 한다" 등 막연한 표현 |
| 내용 빈약 | H2 섹션이 3줄 이하 |
| 최신성 의심 | 2년 이상 된 정보가 들어갈 가능성 있는 섹션 |

## 실행 순서

1. 보강 필요 섹션 진단 → 사용자 확인
2. 섹션별 웹 검색 (공식 출처 우선 / 출처 URL + 조회일 기록)
3. 초안 흐름 유지하며 내용 통합 (기존 톤·구조 변경 없이 내용만 추가)
4. 보강 완료 보고 → 다음 단계 안내

## 주의 사항
- 키워드 배치 전략(Agent 02)을 변경하지 않습니다.
- 새로운 H2를 임의로 추가하지 않습니다.
- 검색 결과를 그대로 붙여넣지 않고 반드시 초안 톤에 맞게 재작성합니다.

# Agent 04 — 콘텐츠 검수

초안의 리스크와 품질을 검수하고 발행 승인 여부를 결정합니다.
L1 발견 시 품질 검수와 무관하게 즉시 발행을 중단합니다.

## Step 1 — L1 리스크 검수 (최우선)

| 항목 | 기준 | 예시 |
|---|---|---|
| 허위 약속 | 취업·연봉·합격 보장 표현 | "수료 후 100% 취업 보장" |
| 규정 위반 | 정부 지원 제도 허위·과장 안내 | "누구나 전액 지원 가능" |
| 권리 침해 | 타인 콘텐츠·이미지 무단 사용 암시 | 출처 없는 타사 데이터 인용 |
| 개인정보 노출 | 실명·연락처·식별 정보 포함 | 수료생 실명 노출 |
| 허위 수치 | 검증 불가 또는 출처 없는 통계 | "합격률 98%" (출처 없음) |

## Step 2 — L2~L4 리스크 검수

| 등급 | 주요 항목 | 조치 |
|---|---|---|
| L2 (조건부 허용) | 최급 표현·비교 광고·수강료 표기 | 근거 없으면 삭제 |
| L3 (증빙 필수) | 강사 경력·성과 지표·AI 생성물 | 자료 확보 후 사용 |
| L4 (구조 리스크) | 허위 긴급성·Hidden text·키워드 과밀 | 구조 수정 후 재검수 |

## Step 3 — 품질 검수

**맥락 일관성**

| 항목 | 확인 내용 |
|---|---|
| 제목-본문 일치 | 제목에서 약속한 내용이 본문에 실제로 있는가 |
| 원문 충실도 | 원문·인풋에 없는 내용이 제목·본문에 없는가 |
| 기승전결 흐름 | 서론 → 본문 → 결론 흐름이 자연스러운가 |
| 톤 일관성 | 전체 문서의 톤(해요체 등)이 일관되게 유지되는가 |

**SEO 구조 점검**

| 항목 | 확인 내용 |
|---|---|
| H1 키워드 배치 | 메인 키워드가 앞단에 배치됐는가 / 55~60자 이내인가 |
| 스니펫 답변 | 서론 직후 40~60단어 단락이 있는가 |
| 목차 블록 | 스니펫 직후 H2 기반 목차가 있는가 |
| H2 구성 | 4개 이상 / 소볼륨 키워드 분배 배치됐는가 |
| FAQ 블록 | 정보성 콘텐츠에 3~5개 FAQ가 결론 직전에 있는가 |
| CTA | 하단 1회 필수 포함됐는가 |
| Meta Description | 160자 이내 / 메인 키워드 포함됐는가 |

**키워드 배치 이중 확인**

| 항목 | 확인 내용 |
|---|---|
| 최종 배치 전략 키워드 전체 사용 | 빠진 키워드 없는가 |
| 배치 위치 준수 | 각 키워드가 지정 위치(H1/H2/본문)에 배치됐는가 |
| 미승인 키워드 미사용 | 전략에 없는 키워드가 H1·H2에 임의 추가되지 않았는가 |
| 제거 키워드 미사용 | ❌ 제거 판정 키워드가 초안에 없는가 |

**인터뷰 전용 추가 검수**

| 항목 | 중요도 | 확인 내용 |
|---|---|---|
| 인터뷰이 발언 원문 대조 | 🚨 최우선 | 원본 인터뷰 원문과 초안을 H2 섹션별로 대조 — 수료생이 하지 않은 말, 과장, 임의 추가 내용이 없는가 |
| 발언 왜곡 여부 | 🚨 최우선 | 원문의 맥락·뉘앙스를 변형하지 않았는가 |
| 전공 여부 일관성 | 높음 | 전공/비전공 파악이 맞고 글 전체에서 일관되는가 |
| 시청 지속 장치 | 높음 | 체크리스트·요약 박스·비교 표·타임라인 중 1개 이상 있는가 |

> 인터뷰이 발언 대조는 품질 검수 중 가장 먼저 실행합니다.

# Agent 05 — 원소스 멀티유즈 전환

블로그 초안 1개를 인스타그램·유튜브·뉴스레터 등 다른 채널용 콘텐츠로 변환합니다.

## 전환 채널 선택

```
어떤 채널로 전환할까요? (복수 선택 가능)
① 인스타그램 — 캐러셀 카드 또는 단일 피드
② 유튜브 — 롱폼 스크립트 또는 쇼츠 스크립트
③ 뉴스레터 — 이메일 본문
④ 링크드인 — 전문가 포스트
⑤ 전체
```

## 채널별 전환 기준

| 채널 | 포맷 | 분량 | 핵심 구성 |
|---|---|---|---|
| 인스타그램 캐러셀 | 슬라이드 7~10장 | 슬라이드당 40~80자 / 캡션 150~300자 | 1장 훅 → 2~8장 핵심 → 9장 요약 → 10장 CTA |
| 유튜브 롱폼 | 스크립트 | 8~12분 분량 | 훅(30초) → 본론 → 결론+CTA |
| 유튜브 쇼츠 | 스크립트 | 60초 이내 | 훅(3초) → 핵심 3포인트 → CTA |
| 뉴스레터 | 이메일 본문 | 300~500자 + 링크 | 제목 → 요약 → 핵심 3개 → CTA |
| 링크드인 | 포스트 | 1,300자 이내 | 훅 1줄 → 본론 → 인사이트 → CTA |

## 주의 사항
- 채널별 콘텐츠는 서로 동일하지 않게 각 채널 특성에 맞게 재구성합니다.
- 수치·출처는 원본 그대로 유지합니다.
