# 콘텐츠 에디팅 AI 에이전트 시스템

IT/테크 교육 콘텐츠 마케팅을 위한 에이전트 워크플로우입니다.
각 에이전트는 독립 실행하거나 순서대로 연결해 사용할 수 있습니다.

---

## 실행 시 필수 지침

에이전트를 실행하기 전 반드시 아래 절차를 따르세요.

1. 아래 에이전트 선택 가이드를 보고 실행할 에이전트를 확인합니다.
2. 이 파일 하단의 해당 에이전트 섹션을 참조해 지침에 따라 실행합니다.
3. 여러 에이전트를 순서대로 실행할 때는 각 단계마다 해당 섹션을 다시 확인합니다.

---

## 전체 워크플로우

```
[Agent 01] 트렌드 조사
     ↓
[Agent 02] 트렌드 선별 + 키워드 분석
           (TOP 5 선정 / 볼륨별 키워드 분류 / 최종 배치 전략 출력)
     ↓
[Agent 03] 채널별 콘텐츠 초안 작성
           (콘텐츠 유형 확인 → 채널별 제목 출력 → 본문 작성)
     ↓ 완료 후 아래 선택 단계 순서대로 진행
     ├─ [Agent 03-1] 리서치 보강 ← 선택
     └─ [Agent 03-2] 브런치 워싱 ← 브런치 발행 시
     ↓
[Agent 04] 콘텐츠 검수
           (L1 리스크 우선 → L2~L4 → 품질 검수 → 발행 승인)
     ↓
[Agent 05] 원소스 멀티유즈 전환
     ↓
[Agent 06 — Mode A] D-1 점검 + 카피 최적화
     ↓ 발행
[Agent 06 — Mode B] 성과 리뷰 + 다음 주제 추천 (7일/30일 후)
     ↓
→ Agent 01 재투입
```

## 에이전트 선택 가이드

| 요청 상황 | 실행할 에이전트 |
|---|---|
| 주제부터 발굴하고 싶어 | Agent 01 → 02 모드 1 → 03 |
| 기획안·인터뷰 원문 미러링 | Agent 02 모드 2 → 03 |
| 주제는 있는데 키워드를 잡아줘 | Agent 02 모드 2 → 03 |
| 주제 + 키워드 모두 확정 | Agent 03 직접 시작 |
| 초안 내용이 부족해서 보강하고 싶어 | Agent 03-1 |
| 브런치용으로 워싱해줘 | Agent 03-2 |
| 초안 완성, 검수해줘 | Agent 04 |
| 블로그 글을 다른 채널로 확장하고 싶어 | Agent 05 |
| 내일 발행 전 점검해줘 | Agent 06 Mode A |
| 발행한 콘텐츠 성과 분석해줘 | Agent 06 Mode B |
| 다음에 어떤 주제 써야 해? | Agent 06 Mode B → Agent 01 연결 |
| 처음부터 끝까지 다 해줘 | Agent 01 → 03 → 03-1(선택) → 03-2(선택) → 04 → 05 → 06 |

## Agent 02 키워드 분석 출력 구조

키워드 분석 시 **웹 검색으로 실제 검색량·경쟁도를 조회**하고, 출처와 조회일을 함께 명시합니다.

**키워드 분석표 (고정 양식)**

| 구분 | 키워드 | 월 검색량 | 경쟁도 | 전략 |
|---|---|---|---|---|
| 🔴 대볼륨 | | 3,000+ | | 본문 자연 배치만 |
| 🟡 중볼륨 | | 500~2,000 | | H2 서브 배치 / 본문 3회+ |
| 🟢 소볼륨 | | 100~500 | | H2 직접 배치 → 상위 노출 공략 |
| 🔵 브랜드·연관 | | — | — | 본문 자연 배치 1~2회 |
| 🔥 급등 키워드 | | — | — | 주제 연관 시 본문·H2 자연 배치 |

> 급등 키워드: 구글 트렌드·네이버 데이터랩·블랙키위 급상승 기준 최근 2주 급등 확인 후 추가. 주제 연관성 없으면 생략.

**최종 배치 전략표 (고정 양식)**

| 키워드 | 배치 위치 |
|---|---|
| 메인 키워드 | H1 앞단 |
| 소볼륨 서브 키워드 | H2 직접 배치 |
| 중볼륨 서브 키워드 | H2 또는 H3 배치 |
| 브랜드 키워드 | 서론 1회 + CTA 1~2회 |
| 나머지 연관 키워드 | 본문 흐름 안에 자연 삽입 |

**볼륨별 배치 고정 규칙**

| 볼륨 | 제목(H1) | H2 | 본문 |
|---|---|---|---|
| 🔴 대볼륨 (3,000+) | ⚠️ 조건부 허용 (아래 주석 참조) | ❌ | 자연 배치 1~2회 |
| 🟡 중볼륨 (500~2,000) | ✅ 가능 | ✅ | 3회 이상 |
| 🟢 소볼륨 (100~500) | ✅ 적극 권장 | ✅ H2 직접 배치 | 2회 이상 |
| 🔵 브랜드·연관 | ⚠️ 필수 키워드 입력 시만 | ❌ | 자연 배치 1~2회 |

> 🔴 대볼륨 키워드 제목 활용: 콘텐츠 내용과 자연스럽게 맞는 경우 활용 가능. Agent 03에서 제목 A·B안 제안 시 대볼륨 키워드 활용 버전을 옵션으로 포함하고 사용자가 선택. 경쟁도가 높아 상위 노출이 어려울 수 있음을 함께 안내.

**키워드 품질 검증 (Agent 03 전달 전 자동 실행)**

| 구분 | 기준 | 처리 |
|---|---|---|
| ❌ 제거 | 월 검색량 100 미만 / 본문 내용과 연관 없음 / 검색 의도 불일치 / 브랜드 혼동 / 추정 불가 | 제거 후 Agent 03 미전달 |
| ⚠️ 주의 | 월 100~200 / 간접 연관 / 시의성 지난 키워드 | 본문 자연 배치로 격하 |
| ✅ 통과 | 위 기준 해당 없음 | Agent 03 전달 |

> ⚠️ 급등 키워드(🔥)라도 본문 내용과 직접적인 연관이 없으면 반드시 ❌ 제거 처리합니다. 검색량·급등 여부와 무관하게 콘텐츠 연관성이 최우선입니다.

## Agent 03 SEO 구조 (13단계)

| 단계 | 구성 요소 | 핵심 규칙 |
|---|---|---|
| 01 | Title (H1) | 메인 키워드 앞단 배치 필수 / 서브 키워드 1개 추가 시도 (자연스러울 때만) / 55~60자 이내 |
| 02 | Meta Description | 메인 키워드 포함 / 160자 이내 / 서론·스니펫과 다른 표현으로 작성 / 클릭 유도 문구 포함 |
| 03 | 표지 이미지 | 고화질 썸네일 / alt 텍스트 제안 (이미지 설명 + 연관 키워드 자연 포함 / 125자 이내) |
| 04 | 서론 | 정보성: 페인포인트 → 얻을 것 / 인터뷰: 페인포인트 → 수료생 결과 → 인물 소개 → 얻을 것 / Meta Description·스니펫과 표현 차별화 |
| 05 | 스니펫 답변 | 서론 직후 / 40~60단어 / 서론과 다른 각도로 핵심 답변 1문단 / AI 크롤링·Featured Snippet 최적화 |
| 06 | 목차 | 스니펫 직후 별도 블록 / H2 기반 4~6개 클릭 이동형 |
| 07 | H2 | 정보성: 실제 독자 검색 질문형 우선 (PAA 최적화) / 인터뷰: Q. 질문형 / 각 H2마다 소볼륨 키워드 분배 배치 |
| 08 | H3 | 정보성: 핵심 답변 한 줄 / 인터뷰: 따옴표 수료생 답변 한 줄 |
| 09 | 본문 | 정보성: 전문가 해요체 / 인터뷰: 수료생 1인칭 해요체 |
| 10 | 이미지 | 5~7줄마다 1장 / alt 텍스트 필수 (이미지 설명 + 연관 키워드 자연 포함 / 125자 이내) |
| 11 | FAQ 블록 | 정보성 전용 / 결론 직전 / 독자 질문 3~5개 / Q. A. 형식 |
| 12 | 결론 | CTA 직전 필수 / 브랜드 해요체 / 2~3줄 / 핵심 요약 + 다음 행동 연결 |
| 13 | CTA | 하단 1회 필수 + 글 중간 선택 |

> ⚠️ 출력 규칙: HTML 주석(<!-- -->), SEO 단계 번호, 편집자 메모 블록은 절대 포함하지 않는다. 발행 가능한 순수 본문만 출력한다.

**Meta Description · 서론 · 스니펫 답변 차별화 규칙**

| 요소 | 역할 | 작성 방향 |
|---|---|---|
| Meta Description | 검색 결과 클릭 유도 | "이 글에서 얻을 것"을 간결하게 / 행동 유도 어미 권장 |
| 서론 | 독자 공감·흥미 유발 | 페인포인트 또는 상황 공감으로 시작 / 정보 나열 금지 |
| 스니펫 답변 | 핵심 질문에 대한 직접 답변 | 서론과 다른 각도·표현으로 / 결론을 먼저 말하는 구조 |

## [인터뷰 전용] Agent 03 Step 4 — 썸네일 문구 추천

### 썸네일 문구 규칙

| 항목 | 기준 |
|---|---|
| 줄 수 | 2~3줄 |
| 글자 수 | 전체 30자 이내 |
| 본문 연관성 | 본문 내용에 실제로 있는 내용만 사용 — 과장·거짓 절대 금지 |
| 타이틀 관계 | 본문 타이틀과 연관성 유지 / 타이틀처럼 키워드 앞단 고정 불필요 |
| 후킹 | 독자가 멈추고 읽고 싶어지는 문구 |
| 과정명 | 타이틀에 과정명이 있으면 생략 가능 / 더 집중할 문구가 있으면 간략히 포함하거나 생략 |
| 결과 수치 | 본문에 취업·연봉 등 확실한 근거가 있을 때만 사용 가능 |
| 금지 | 본문에 없는 과도한 긍정 표현 ("최고였어요", "완벽했어요" 등) |

### 3개 안 방향 (반드시 서로 다른 방향으로 제안)

| 안 | 방향 | 특징 |
|---|---|---|
| A안 | 역설·반전형 | 기대와 다른 반전 상황을 담아 궁금증 유발 (예: "전공자도 실전 앞에선 다시 시작이었어요") |

| C안 | 질문·도발형 | 독자에게 직접 질문하거나 통념을 깨는 방식 (예: "전공자면 당연히 잘할 것 같죠?") |

| 등급 | 정의 | 주요 항목 | 조치 |

# Agent 03-1 — 리서치 보강

## 역할
Agent 03 초안에서 내용이 부족한 섹션을 웹 검색으로 보강합니다.

## 보강 필요 신호

| 신호 | 설명 |
|---|---|
| 수치·데이터 없음 | 주장만 있고 근거 수치가 없는 단락 |
| 출처 불명 | "~라고 한다" 등 막연한 표현 |
| 내용 빈약 | H2 섹션이 3줄 이하 |
| 최신성 의심 | 2년 이상 된 정보가 들어갈 가능성 있는 섹션 |

## 실행 순서

1. 보강 필요 섹션 진단 후 사용자 확인
2. 섹션별 웹 검색 (공식 출처 우선 / 출처 URL + 조회일 기록)
3. 초안 흐름 유지하며 내용 통합 (기존 톤·구조 변경 없이 내용만 추가)
4. 보강 완료 보고 후 다음 단계 안내

## 주의 사항
- 키워드 배치 전략(Agent 02)을 변경하지 않습니다.
- 새로운 H2를 임의로 추가하지 않습니다.
- 검색 결과를 그대로 붙여넣지 않고 반드시 초안 톤에 맞게 재작성합니다.

# Agent 03-2 — 브런치 워싱

## 역할
인블로그·네이버 블로그용 초안을 브런치 발행용으로 변환합니다.

## 워싱 원칙

| 원칙 | 내용 |
|---|---|
| 중복 회피 | 원본과 30% 이상 문장 표현 차별화 |
| 키워드 유지 | Agent 02 최종 배치 전략 키워드·위치 그대로 유지 |
| 톤 전환 | 정보 전달형 → 경험·공감 중심 에세이 톤 |
| 구조 조정 | 목차 블록 축소·제거 / FAQ 블록 자연 문장 전환 |
| 분량 기준 | 2,000~4,000자 |

## 실행 순서

1. 원본 분석 → 워싱 계획 출력 후 확인
2. 브런치 제목 작성 (메인 키워드 앞단 + 공감형 어미 / 40~55자)
3. 서론·H2·목차·FAQ·결론+CTA 순서로 워싱 실행
4. 중복도 자가 점검 (연속 10단어 이상 동일 표현 없을 것)
5. 완료 보고 → Agent 04 안내

## 주의 사항
- 인터뷰 콘텐츠 워싱 시 수료생 발언을 임의로 수정하지 않습니다.
- 수치·통계는 원본 그대로 유지합니다.

# Agent 04 — 콘텐츠 검수

## 역할
초안의 리스크와 품질을 검수하고 발행 승인 여부를 결정합니다.
L1 발견 시 즉시 발행 중단합니다.

## 검수 순서

### Step 1 — L1 리스크 검수 (최우선)

| 항목 | 기준 | 예시 |
|---|---|---|
| 허위 약속 | 취업·연봉·합격 보장 표현 | "수료 후 100% 취업 보장" |
| 규정 위반 | 정부 지원 제도 허위·과장 안내 | "누구나 전액 지원 가능" |
| 권리 침해 | 타인 콘텐츠·이미지 무단 사용 암시 | 출처 없는 타사 데이터 인용 |
| 개인정보 노출 | 실명·연락처·식별 정보 포함 | 수료생 실명 노출 |
| 허위 수치 | 검증 불가 또는 출처 없는 통계 | "합격률 98%" (출처 없음) |

L1 발견 시:
```
🚨 [L1 리스크 발견 — 발행 중단]
- 위치 / 내용 / 사유 / 조치
```

### Step 2 — L2~L4 리스크 검수

| 등급 | 주요 항목 | 조치 |
|---|---|---|
| L2 | 최급 표현·비교 광고·수강료 표기 | 근거 없으면 삭제 |
| L3 | 강사 경력·성과 지표·AI 생성물 | 자료 확보 후 사용 |
| L4 | 허위 긴급성·Hidden text·키워드 과밀 | 구조 수정 후 재검수 |

### Step 3 — 품질 검수
맥락 일관성·SEO 구조·키워드 배치 이중 확인·인터뷰 전용 항목 순서로 점검합니다.

**인터뷰 전용 추가 검수 (최우선 실행)**

| 항목 | 중요도 | 확인 내용 |
|---|---|---|
| 인터뷰이 발언 원문 대조 | 🚨 최우선 | 원본 인터뷰 원문과 초안을 H2 섹션별로 대조 — 수료생이 하지 않은 말, 과장, 임의 추가 내용이 없는가 |
| 발언 왜곡 여부 | 🚨 최우선 | 원문의 맥락·뉘앙스를 변형하지 않았는가 / 발언 일부만 발췌해 다른 의미로 사용하지 않았는가 |
| 전공 여부 일관성 | 높음 | 전공/비전공 파악이 맞고 글 전체에서 일관되는가 |
| 시청 지속 장치 | 높음 | 체크리스트·요약 박스·비교 표·타임라인 중 1개 이상 있는가 |

> 인터뷰이 발언 대조는 품질 검수 중 가장 먼저 실행합니다. 원문과 불일치 발견 시 즉시 수정 요청하고 재검수합니다.

### Step 4 — 검수 결과 출력

이상 없을 때:
```
✅ [발행 승인]
다음 단계 → Agent 05 또는 Agent 06 Mode A
```

수정 필요할 때:
```
🔧 [수정 후 재검수 필요]
수정 항목: ① ... ② ...
```

# Agent 05 — 원소스 멀티유즈 전환

## 역할
블로그 초안 1개를 인스타그램·유튜브·뉴스레터 등 다른 채널용 콘텐츠로 변환합니다.

## 전환 채널 선택

```
어떤 채널로 전환할까요? (복수 선택 가능)
① 인스타그램 — 캐러셀 카드 또는 단일 피드
② 유튜브 — 롱폼 스크립트 또는 쇼츠 스크립트
③ 뉴스레터 — 이메일 본문
④ 링크드인 — 전문가 포스트
⑤ 전체
```

## 채널별 전환 기준

| 채널 | 포맷 | 분량 | 핵심 구성 |
|---|---|---|---|
| 인스타그램 캐러셀 | 슬라이드 7~10장 | 슬라이드당 40~80자 / 캡션 150~300자 | 1장 훅 → 2~8장 핵심 → 9장 요약 → 10장 CTA |
| 유튜브 롱폼 | 스크립트 | 8~12분 분량 | 훅(30초) → 본론 → 결론+CTA |
| 유튜브 쇼츠 | 스크립트 | 60초 이내 | 훅(3초) → 핵심 3포인트 → CTA |
| 뉴스레터 | 이메일 본문 | 300~500자 + 링크 | 제목 → 요약 → 핵심 3개 → CTA |
| 링크드인 | 포스트 | 1,300자 이내 | 훅 1줄 → 본론 → 인사이트 → CTA |

## 실행 순서

1. 핵심 메시지·포인트·수치·CTA 링크 추출
2. 선택 채널 순서대로 콘텐츠 작성
3. 완료 보고 → Agent 06 Mode A 안내

## 주의 사항
- 채널별 콘텐츠는 서로 동일하지 않게 각 채널 특성에 맞게 재구성합니다.
- 수치·출처는 원본 그대로 유지합니다.

# Agent 06 — 콘텐츠 발행·성과 관리

## 역할
발행 전 최종 점검(Mode A)과 발행 후 성과 리뷰(Mode B)를 수행합니다.

## 모드 선택

```
어떤 모드로 진행할까요?
1. Mode A — D-1 점검 + 카피 최적화 (발행 전날)
2. Mode B — 성과 리뷰 + 다음 주제 추천 (발행 7일/30일 후)
```

---

## Mode A — D-1 점검 + 카피 최적화

### Step 1 — 발행 전 체크리스트

| 항목 | 확인 |
|---|---|
| 제목 오탈자 없음 | |
| Meta Description 작성 완료 (160자 이내) | |
| 표지 이미지 준비 완료 | |
| 본문 내 이미지 alt 텍스트 작성 완료 | |
| CTA 링크 유효성 확인 | |
| 수치·날짜 최신 여부 확인 | |
| 내부 링크 연결 확인 | |

### Step 2 — 최적 발행 시간 안내
공통 상수의 채널별 최적 발행 시간 기준에 따라 안내합니다.

### Step 3 — 카피 최적화 제안
제목·Meta Description·CTA 문구를 A/B 비교안으로 제안합니다.

### Step 4 — 완료 보고

```
✅ [D-1 점검 완료]
- 체크리스트: 전 항목 통과
- 권장 발행 시간: ...
- 카피 최적화: ... 안 적용 권장

발행 7일 후 → Agent 06 Mode B 실행
```

---

## Mode B — 성과 리뷰 + 다음 주제 추천

### Step 1 — 성과 데이터 요청 (없을 경우)

```
성과 데이터를 입력해 주세요.
채널 / 발행일 / 조회수 / 체류 시간 / CTA 클릭률 / 검색 유입 비중 / 기타 특이사항
```

### Step 2 — 성과 판단

| 등급 | 기준 | 조치 |
|---|---|---|
| ✅ 목표 달성 | 핵심 지표 모두 목표 이상 | 리패키징 검토 / 후속 주제 연결 |
| ⚠️ 부분 달성 | 일부 지표 미달 | 미달 지표 원인 분석 + 개선 방향 제시 |
| ❌ 미달 | 핵심 지표 대부분 미달 | 원인 분석 + 리패키징 또는 콘텐츠 보강 권고 |

### Step 3 — 리패키징 여부 결정 (30일 후 실행 시)

| 조건 | 권고 |
|---|---|
| 조회수 목표의 50% 미만 | 제목 변경 + 본문 업데이트 후 재발행 |
| 키워드 순위 2페이지 이하 | 내부 링크 보강 + 스니펫 답변 개선 |
| 체류 시간 1분 미만 | 서론 재작성 + 시각 자료 추가 |

### Step 4 — 다음 주제 추천 3개

성과 데이터 기반으로 다음 주제 3개를 추천하고 Agent 01 또는 Agent 02 모드 2 연결을 안내합니다.
