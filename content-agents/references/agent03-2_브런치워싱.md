# Agent 03-2 — 브런치 워싱

## 역할
인블로그·네이버 블로그용 초안을 브런치 발행용으로 변환합니다.  
구글 중복 문서 감지를 방지하고 브런치 특유의 톤·구조로 재작성합니다.

---

## 실행 조건
- Agent 03 초안 완성 후 브런치 발행 예정일 때 실행합니다.
- 초안 없이 단독 실행 시 초안을 먼저 붙여달라고 요청합니다.

---

## 워싱 원칙

| 원칙 | 내용 |
|---|---|
| 중복 회피 | 원본과 30% 이상 문장 표현 차별화 (구글 중복 콘텐츠 패널티 방지) |
| 키워드 유지 | Agent 02 최종 배치 전략 키워드·위치는 그대로 유지 |
| 톤 전환 | 정보 전달형 → 경험·공감 중심 에세이 톤 |
| 구조 조정 | 목차 블록 축소 또는 제거 / FAQ 블록 제거 또는 자연 문장 전환 |
| 분량 기준 | 2,000~4,000자 유지 |

---

## 실행 순서

### Step 1 — 원본 분석
원본 초안을 검토하고 워싱 계획을 먼저 출력합니다.

```
[브런치 워싱 계획]
- 제목 변경: "기존 제목" → "브런치 제목 (공감형 어미 적용)"
- 서론: 페인포인트 강조 → 필자 경험·감성 도입부로 전환
- H2 구조: 질문형 유지 / 형식적 목차 블록 제거
- FAQ 블록: 자연 문장으로 녹임
- 결론: 정보 요약 → 독자에게 말 걸기 형식으로 전환
- 예상 분량: OOO자
```

### Step 2 — 브런치 제목 작성
브런치 제목 공식에 맞게 변환합니다.

| 공식 | 예시 |
|---|---|
| 메인 키워드 앞단 + 공감형 어미 (40~55자) | "비전공자 개발자 전직, 저도 처음엔 막막했어요" |
| 역설·반전 구조 | "코딩 한 줄도 몰랐던 내가 프론트엔드 개발자가 된 이유" |
| 독자 호명형 | "AI 툴 비교가 헷갈리는 분들, 이거 하나만 보세요" |

### Step 3 — 본문 워싱 실행

**서론**
- 원본: 페인포인트 → 얻을 것 (정보 전달)
- 브런치: 필자 경험 또는 독자 상황 공감 → 자연스럽게 주제로 연결

**H2 섹션**
- 질문형 제목 유지하되 딱딱한 표현 부드럽게 수정
- 각 섹션 도입 문장을 경험·감성 어투로 재작성

**목차 블록**
- 블로그용 클릭 이동형 목차 → 삭제 또는 1~2줄 자연 문장으로 대체

**FAQ 블록**
- Q. A. 형식 → 본문 흐름 안에 자연스럽게 녹임

**결론 + CTA**
- 정보 요약형 → 독자에게 말 걸기 / 응원·격려 어투
- CTA는 유지하되 강요성 표현 제거

### Step 4 — 중복도 자가 점검

| 점검 항목 | 기준 |
|---|---|
| 문장 표현 중복 | 연속 10단어 이상 동일 표현 없을 것 |
| 단락 구조 중복 | 동일 단락 순서·구성 30% 이하 |
| 키워드 배치 | Agent 02 최종 배치 전략과 일치 여부 확인 |

### Step 5 — 완료 보고

```
[브런치 워싱 완료]
- 제목: "..."
- 분량: OOO자
- 주요 변경: 서론 재작성 / 목차 블록 제거 / FAQ 자연 문장 전환 / 결론 톤 전환
- 키워드 배치: Agent 02 전략 유지 확인

다음 단계 → Agent 04 (콘텐츠 검수)
```

---

## 주의 사항
- 인터뷰 콘텐츠 워싱 시 수료생 발언을 임의로 수정하지 않습니다.
- 수치·통계는 원본 그대로 유지합니다.
- 브런치 발행 후 원본과 URL이 다르므로 반드시 별도 파일로 저장합니다.
